# _smoke PRD (린트 검증용 더미)

## 1. 배경과 문제 정의
스모크 테스트용 더미 PRD다. 도구(`lint-prd.js`)가 11개 섹션과 깊이를 제대로 보는지 확인하려고 쓴다.

## 2. 목표 및 성공 지표
`node scripts/lint-prd.js _smoke`가 green(섹션 11/11)을 반환한다. 측정: exit 0.

## 3. 타깃 사용자와 사용 시나리오
이 하네스를 처음 돌려보는 운영자. 첫 실제 프로젝트 전에 도구가 동작하는지 확인한다.

## 4. 기능 요구사항
- (MVP) 린트가 섹션 부재를 error로 잡는다 — 사용자가 빈 섹션으로 /design에 넘기면 입력이 빈다.
- (다음) 깊이 warn(관계표기·다크모드·뷰포트)을 안내한다.
우선순위 근거: 부재는 진입 차단(error), 깊이는 권고(warn).

## 5. 데이터 모델
핵심 엔터티: Project, Stage. 관계: Project 1:N Stage (한 프로젝트가 여러 단계를 가진다).
물리 스키마(컬럼 타입·인덱스·마이그레이션) 확정은 범위 밖 — 개발 단계의 결정사항이다.

## 6. 사용자 흐름
운영자가 PRD를 쓴다 → `lint-prd`를 돌린다 → green이면 `/design`으로 넘어간다.

## 7. 화면 목록 및 화면별 요구사항
- 홈(index): 목록·상세로 이동.
- 목록(list): 토글·카운터·상세 링크.
- 상세(detail): 홈/목록으로 복귀.

## 8. 디자인 방향
톤: 중립·기능적. 레퍼런스: 기본 HTML. 대상 뷰포트: 데스크톱 1280px. 다크모드: 없음(라이트만).

## 9. 비기능 요구사항
린트는 외부 의존성 없이 순수 Node로 돈다. 접근성: 토큰 대비비 WCAG AA.

## 10. 범위 외 (Out of Scope)
실제 제품 기능, 인증, 서버. 이건 도구 검증용 더미다.

## 11. 미해결 이슈와 가정
- 가정: 운영자가 Node 18+ 환경을 가진다.
- 미해결: 없음(더미라 추가 검증 불필요).
