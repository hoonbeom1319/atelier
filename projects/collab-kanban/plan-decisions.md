# collab-kanban — plan 의사결정 근거 (시장조사 기록)

> /forge(무인) §B. market-researcher 에이전트(웹검색)가 plan 질문을 축으로 나눠 병렬 조사하고, 각 답을 출처 URL로 근거화한 기록이다.
> **이 문서는 PRD §11(가정)의 뒷받침 아티팩트다.** §11-B의 "근거 있는 결정"(가정 → 결정 승격)은 아래 grounded 행에 대응한다 — confidence가 `grounded`인 항목은 출처가 있어 PRD 본문에 "근거 있는 결정"으로 반영했고, `guess`인 항목은 PRD §11-D에 추측으로 남겨 본문에서 단정하지 않았다.
> 형식: 질문 → 근거(출처) → 답/권고 → confidence(grounded/guess) → §11 교차참조.

조사 3축: ① 경쟁/유사 칸반 협업도구(Trello·Linear·Jira·Notion·Asana)의 핵심기능·차별점·MVP 경계 / ② 칸반 사용자 불만·니즈·협업 워크플로우(담당자·댓글·필터) 실사용 패턴 / ③ 칸반/이슈트래커 데이터 모델 관례와 흔한 엣지케이스.

## 축 1 — 경쟁/유사 칸반 협업도구의 핵심기능·차별점·MVP 경계

**요약.** 경쟁 도구(Trello·Linear·Jira·Notion·Asana) 조사 결과, 사용자가 정한 4개 MVP 협업 기능은 경쟁 도구의 '카드 칸반' 핵심과 1:1로 일치하며 단일 보드 MVP 경계로 타당함(충돌 없음). 포지션은 'Trello의 진입 단순함 + Linear의 절제된 비주얼' 교집합 — 디자인 공은 (1)카드 표면 정보밀도 절제 (2)드래그앤드롭 체감 (3)멤버 협업 가시성에 들어가야 함. 데이터 모델은 Board→Column(순서)→Card(컬럼 내 순서·다중 담당자·다중 라벨)+Member·Label·Comment·Activity가 표준. 댓글/활동로그는 카드 상세 패널 하단 타임라인 관례. 비주얼은 Linear가 실제로 base+accent+contrast로 라이트/다크 생성 → 촘촘한 중립 그레이 램프(50~900)+단일 accent semantic 토큰+다크 별도 페어 스케일 권고, 그레이끼리 대비비 WCAG 위험으로 a11y 게이트 필수.

## 축 2 — 칸반 사용자 불만·니즈·협업 워크플로우 실사용 패턴

**요약.** 소규모 팀 칸반의 실사용 불만은 기능 부족보다 "디테일 부재"에 몰린다 — 카드 과부하(WIP 미관리)로 보드가 시각적으로 무너지고, 댓글이 plain-text라 @멘션·첨부가 안 되며, 모든 댓글 알림으로 알림 피로가 생긴다. 사용자가 정한 MVP 4개 기능은 정확히 핵심 통점을 정조준(충돌 없음). 디자인 권고: (1) 카드 앞면=제목·담당자 아바타·마감일·라벨 1~2개·우선순위 신호만(나머지 디테일 패널), (2) 드래그 시 drop-zone 하이라이트+lift/shadow+빈 컬럼 빈상태, (3) 댓글은 @멘션 전제·멘션된 사람만 알림(알림 피로 방지), (4) 활동 로그는 댓글과 분리(탭/스트림), (5) Linear류 톤=4px 그리드·40~60% 불투명 그레이 텍스트·풀채도는 상태/우선순위/인터랙션에만·다크 4단 표면 elevation. 엣지케이스(미배정·overdue·빈 컬럼·WIP 과부하)는 모두 디자인 상태(state)로 표현해야 함.

## 축 3 — 칸반/이슈트래커 데이터 모델 관례와 흔한 엣지케이스

**요약.** Trello·Linear의 공개 데이터 모델 근거로 단일 보드 MVP 엔티티 관계를 권고. 핵심 계층=Board→Column(List)→Card의 1:N, Card↔Member(담당자)·Card↔Label은 N:M, Activity/Comment는 Card에 매달린 시간순 append-only 로그(Trello가 댓글을 Action 특수형으로 둠). 카드 정렬은 정수 시퀀스가 아니라 부동소수 'pos'(분수 인덱싱) — 인접 두 카드 중간값을 부여해 재정렬 시 다른 행 안 건드리는 게 업계 관례. 마감일=ISO8601+dueComplete 불리언, 라벨=색+이름. 엣지케이스 우선순위=빈 컬럼/미배정/긴 제목 오버플로/지난 마감/긴 컬럼 스크롤이며 모두 명시적 빈/경계 상태로 그려야 함. 사용자 답(단일 보드·실시간/인증 범위 밖·데스크톱 우선)과 충돌하는 결과 없었고, WIP 한도는 칸반 정석이나 MVP 범위에 없어 '선택(다음)'.
