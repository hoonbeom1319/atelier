## §디자인 범위 확장 (사용자 피드백 2026-06-26 — PRD 원안보다 넓힘)
> 초기 산출(보드 단일 화면)이 "서비스로 안 보인다"는 피드백 → **전체 서비스 형태**로 확장: **앱 셸(사이드바+탑바) + 대시보드 홈 + 보드**, 비주얼은 **더 화려·대담하게**.
> PRD §10이 멀티보드/워크스페이스를 범위 밖으로 뒀으나, 여기선 **셸·홈을 "디자인 표현"으로만** 도입한다(멀티보드 실기능·실인증은 여전히 범위 밖 — 사이드바 멤버/설정·마케팅 보드·+새로는 `[범위 밖]` 스텁). 보드의 기능·검증 계약은 그대로 보존하고 그 위에 셸을 덧입힌다.

## §앱 셸 (두 화면 공유 — 서비스 chrome)
- **사이드바(좌, 212px):** 브랜드 + 네비(🏠 홈 → index.html · 📋 보드 → board.html · ▸ 제품(현재 보드) · 👥 멤버`[범위 밖]` · ⚙ 설정`[범위 밖]`). 현재 화면 항목은 링크가 아니라 `aria-current="page"` 비활성 표시(죽은 링크 방지).
- **탑바(상):** 검색 input(`검색`) · `+ 새로`(`[범위 밖]`) · 🔔 알림(드롭다운, `aria-expanded`) · 테마 전환(`aria-pressed`) · 사용자 메뉴 아바타(드롭다운, `aria-expanded`). 보드 화면 탑바엔 **역할 전환**(`aria-pressed`)도. 드롭다운 메뉴 항목(프로필·로그아웃)은 `[범위 밖]`.

## §화면 목록
1. **index.html — 홈 대시보드(앱 셸 안).** 폴더 entry. 셸 + 인사("안녕하세요, 김지원님") + 위젯 **내게 배정된 카드**(→board 링크) · **최근 활동**(시스템 활동 요약) · **내 보드**(제품→board.html / 마케팅 `[범위 밖]`). 진입 후 사이드바 "보드"로 들어간다.
2. **board.html — 제품 보드(앱 셸 안, 핵심 기능 화면).** PRD SC1 그대로 — 셸 + 탑바 아래 필터바·컬럼·카드. 카드 상세(SC2)는 이 화면 위 **오버레이 패널**. 이동/배정/댓글/필터/테마/역할 전부 이 화면 안 인라인 JS.

> 두 화면 모두 같은 셸을 두른 **단일 보드 서비스**다(보드 라우트는 1개 — 제품). index=홈 대시보드, board=보드. 카드 상세는 별도 화면이 아니라 board 위 오버레이. 셸의 멀티보드/멤버/설정은 디자인 표현용 스텁(`[범위 밖]`).

## §사용자 플로우 (A→B·분기)
```
[index] "보드 열기 →" 클릭 ──nav──> [board]

[board] 진입: 헤더(보드명·역할 토글·테마 토글) + 필터바(아바타 스택/라벨/마감/내 카드만) + 컬럼 5개(헤더 이름+카드수, 카드 리스트, "+ 카드 추가")
  · 카드 이동(F1) — ⓐ 네이티브 드래그(마우스): 카드 잡기 → 들림(lift)·플레이스홀더, 유효 컬럼 드롭존 하이라이트 → 놓은 위치 삽입
                    ⓑ 카드 메뉴(키보드/결정적): 카드 "⋮ 카드 메뉴" → "○○로 이동" → 대상 컬럼 끝에 삽입
        └ 공통 결과: 카드 columnId 변경 · 양 컬럼 카운터 갱신 · 활동 로그 'moved' 1건 추가
  · 카드 클릭(F4) ──> [상세 패널 오버레이: 제목·설명·담당자·라벨·마감 + 하단 통합 타임라인(댓글+활동)]
        → 담당자 배정/변경 ──> 카드 표면 아바타 반영 + 활동 'assigned'
        → @멘션 댓글 작성(trim 빈 값 무시) ──> 타임라인에 추가(멘션 칩) · 카드 표면 댓글수 +1
        → 타임라인 필터(전체/댓글/활동) ──> 해당 종류만 표시
  · "+ 카드 추가"(컬럼 하단) ──(제목 trim 비면 무시)──> 컬럼 끝에 카드 추가 · 카운터 +1 · 활동 'created'
  · 필터(F3) — 아바타 토글(멀티)·라벨·마감(오늘/이번주/지남)·"내 카드만" ──> 조건 매칭 카드만, 활성 필터 강조, 컬럼 카운트=필터된 수
        └ (매칭 0건) ──> 보드 빈 상태("조건에 맞는 카드가 없습니다") + 필터 해제 버튼
  · 역할 토글(F8) 편집자 ⇄ 뷰어 ──(뷰어)──> 카드 드래그 불가(draggable 해제)·카드 메뉴/+추가 숨김·"읽기 전용" 안내·상세 패널 댓글 입력 숨김
  · 테마 토글(F9) ──> 라이트 ⇄ 다크(aria-pressed 반전, semantic 토큰 분기) · 선택 영속(localStorage)
[새로고침/재진입] ──> 마지막 **테마** 복원. 역할은 편집자로, 보드 카드 상태는 시드로 재시작(목업 — 결정적 검증을 위해 역할·보드는 비영속)
```

## §화면별 구성요소·상태  ← (2층 기능 검증 spec이 여기서 파생)

### index.html (홈 대시보드 — 앱 셸 안)
- **구성:** 앱 셸(사이드바+탑바) + 인사 헤딩 + 위젯 3종. **내게 배정된 카드**(현재 사용자=김지원의 카드, 각 행 → board.html 링크) · **최근 활동**(시스템 활동 요약, 정적) · **내 보드**(제품 카드 → board.html / 마케팅 `[범위 밖]`).
- **인터랙티브(전부 wired or `[범위 밖]`):** 사이드바 보드/제품 링크(nav) · 배정 카드 링크(nav) · 제품 보드 링크(nav) · 탑바 알림·사용자 드롭다운(`aria-expanded`) · 테마 전환(`aria-pressed`) · (스텁) 멤버·설정·+새로·마케팅. 검색 input. 죽은 컨트롤 0.
- **상태:** 테마 light/dark. 드롭다운 열림/닫힘.

### board.html (메인 — 핵심) — PRD SC1+SC2+SC3
구성 영역:
1. **헤더/툴바** — 보드명, **역할 토글**(편집자/뷰어, `aria-pressed`), **테마 토글**(`aria-pressed`로 light/dark).
2. **필터바** — ⓐ **아바타 스택**(멤버별 토글, 멀티선택, `aria-pressed`; 4명 표시 + "+1" 오버플로) ⓑ **"내 카드만"** 빠른 토글 ⓒ **라벨 필터**(라벨 토글) ⓓ **마감 필터**(오늘/이번주/지남 — 단일 선택 세그먼트) ⓔ 활성 필터 있을 때 **"필터 해제"**.
3. **컬럼 영역** — 가로 배치 컬럼 5개. 각 컬럼: 헤더(`이름` + `카드 수`) + 카드 리스트(세로 스크롤) + 하단 **"+ 카드 추가"**(인라인 입력 토글). 컬럼 폭 초과 시 보드 가로 스크롤(G5).
4. **카드(컴팩트, 보드용)** — 표면: 제목(2줄 말줄임) · 담당자 아바타(미배정=placeholder) · 라벨 칩 1~2개 · 마감일(overdue 경고) · 댓글수(아이콘+숫자) · **"⋮ 카드 메뉴"**(이동). 그 이상 노출 금지.
5. **드래그&드롭 상태(변형 — G2)** — `dragging`(들림) · `drop-target`(드롭존 하이라이트) · 빈 컬럼 드롭존 힌트. 네이티브 DnD 핸들러로 클래스 토글(정적 카드만 그리는 stateInert 0).
6. **카드 상세 패널(SC2, 오버레이)** — 제목·설명·담당자(배정/변경)·라벨·마감 + 하단 **통합 타임라인**(댓글+활동) + **타임라인 필터(전체/댓글/활동)** + **댓글 입력(@멘션 칩 전제)** + 닫기. 뷰어면 댓글 입력 숨김.
7. **빈/경계 상태(SC3 — G3)** — 빈 컬럼(검토) 드롭존 · 필터 0건 보드 빈 상태 · 미배정 placeholder · overdue 강조 · 아바타 +N · 카드 과밀 세로 스크롤 · 읽기 전용 안내.

**컴포넌트별 상태:**
- 카드: 기본 / hover(lift 신호) / dragging / 담당자 있음·미배정 / overdue / 긴 제목 말줄임.
- 컬럼: 카드 있음 / 빈(드롭존) / 드롭 타깃 하이라이트 / 과밀(스크롤).
- 필터 토글(아바타·라벨·내 카드만): 선택(`aria-pressed=true`) / 비선택. 마감 세그먼트: 선택(`aria-current`) / 비선택.
- 역할 토글: 편집자(`aria-pressed=false`) / 뷰어(`aria-pressed=true`).
- 테마 토글: light(`aria-pressed=false`) / dark(`aria-pressed=true`).
- 상세 패널: 닫힘 / 열림(오버레이). 타임라인 필터: 전체 / 댓글 / 활동.
- 보드: 카드 있음 / 필터 매칭 0(빈 상태).

## §인터랙션 모델 (구조 — 모양 아님)
- **카드 이동 = 두 경로, 한 결과.** 마우스=네이티브 HTML5 DnD(시각 상태), 키보드/결정적=카드 메뉴의 "○○로 이동". 둘 다 columnId·position 변경 + 카운터 + 활동 'moved'. (검증 spec은 결정적 메뉴 경로로 이동을 확인하고, DnD 시각 상태는 dragstart/dragenter 디스패치로 클래스 토글을 확인 — 네이티브 DnD 마우스 시뮬레이션의 flaky를 피한다.)
- **담당자 = 카드당 1명(+미배정).** 상세 패널에서 배정/변경. 모델은 다중 확장 비파괴(§5) — 필터 아바타 스택 "+N"으로 다중 표현 변형.
- **필터 = 파생 뷰 상태**(카드 데이터 아님). 아바타·라벨·"내 카드만" = 멀티 토글(상호 누적, `aria-pressed`), 마감 = 단일 선택 세그먼트. 보드는 항상 `(전체 카드) ∩ (활성 필터)`로 렌더, 컬럼 카운트=필터된 수.
- **타임라인 = 통합 단일 스트림**(댓글+활동, 시간 역순) + 종류 필터(전체/댓글/활동). **탭 분리 아님**(C1 → 통합으로 확정: 소규모·미니멀).
- **댓글 = @멘션 전제 단일 입력**(trim 빈 값 무시). 멘션은 body 텍스트 → Member 파싱(별도 참조 없음, §5). 카드 표면은 댓글수(total)만, "미해결" 구분 없음(P1).
- **역할 = 2상태 토글**(편집자/뷰어). 뷰어 = 순수 읽기 전용(드래그·메뉴·생성·편집·배정·댓글 입력 전부 차단/숨김) — **C2 → 뷰어 댓글 불가로 확정**(읽기 전용 의미 명확화).
- **테마 = 2상태 토글**(light/dark), 영속(localStorage). 초기값 light(결정성 — prefers-color-scheme 미추종, C4 → 토글·양모드 렌더·양모드 a11y만 보증).
- **추가 = 컬럼별 인라인 입력**(제목 trim 빈 값 무시). 생성 위치=컬럼 끝.

## §범위 표시
- **범위 안(이번 와이어/하이파이에 만든다):** 앱 셸(사이드바 네비·탑바 검색/알림/사용자/테마) · 홈 대시보드(배정 카드·최근 활동·내 보드) · F1 카드 이동(DnD+메뉴) · F2 담당자 배정·아바타 · F3 필터(멤버/라벨/마감/내 카드만) · F4 상세 패널·통합 타임라인·@멘션 댓글 · F5 카드 추가 · F6 라벨·마감(overdue) · F7 컬럼 카운터 · F8 뷰어 읽기전용 · F9 테마 토글 · 빈/경계 상태 전부.
- **범위 밖(만들지 않음 — `[범위 밖]` 스텁으로만 표현):** 멀티보드/워크스페이스 실전환(사이드바 멤버·설정·마케팅 보드·+새로는 스텁) · 검색 실동작 · 알림/사용자 메뉴 실액션(프로필·로그아웃) · WIP 한도 · 다중 담당자 UI · 삭제 실행취소/아카이브 · 우선순위 마커 · 알림 센터 · 댓글 해결(resolve) · 체크리스트/서브태스크/스윔레인/추정치/간트/첨부/의존성(PRD §10). → 스텁은 `data-scope="out"`+`[범위 밖]` 라벨로 죽은 컨트롤 게이트에서 면제(scope 부풀림 방지).

## §열린 질문 (무인 — 가정으로 닫음, PRD §11 C 항목 확정)
- **C1 타임라인:** 통합 타임라인 + 전체/댓글/활동 필터로 **확정**(탭 분리 아님).
- **C2 뷰어 댓글:** 뷰어는 **순수 읽기 전용 → 댓글 입력도 숨김**으로 확정.
- **C3 긴 제목:** **2줄 말줄임(ellipsis)** 으로 확정(PRD §7 SC1.3). 라벨 길면 칩 내 말줄임.
- **C4 테마 영속/시스템선호:** 초기 **light 고정**(결정성), 선택은 **localStorage 영속**. prefers-color-scheme 미추종. 양모드 렌더·a11y만 보증.
- **DnD 검증 전략:** 네이티브 DnD 마우스 시뮬레이션은 flaky → spec은 **카드 메뉴 이동(키보드 경로 = a11y 요구 §9)** 으로 이동을 결정적으로 확인하고, 시각 상태는 dragstart/dragenter 디스패치로 클래스 토글 확인. 두 경로 모두 와이어드(죽은 컨트롤 0).
- 그 외 미답은 PRD §11 가정 원장에 있음(개념적 멀티유저·단일보드·목업 데이터).
