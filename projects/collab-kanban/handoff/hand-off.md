# collab-kanban — 인계 브리프 (hand-off)

> 이 파일을 받는 쪽 Claude Code 프로젝트에 **그대로 붙여넣고** 시작하세요. 대상 프로젝트의 Claude가 이 브리프를 읽고 **자기 프레임워크·컨벤션으로** 구현합니다. atelier는 대상 코드를 직접 짜지 않습니다(특정 스택에 묶이지 않음).
> 산출물은 외부 의존성 없는 독립 실행 HTML 시안입니다(상태/드래그용 인라인 JS 포함). 구현 시 대상 스택(React/Vue/Svelte 등)으로 옮기되 **아래 계약과 데이터 모델을 보존**하세요.

## 무엇을 만드는가
소규모 팀(3~10명)을 위한 **협업 칸반 서비스**. **앱 셸(좌측 사이드바 네비 + 상단 탑바) 안에 ① 홈 대시보드 ② 제품 보드** 두 화면이 든다. 보드에서 카드를 컬럼 간 드래그&드롭으로 옮기고, 사람에게 배정하고(아바타·필터), 카드 안에서 @멘션 댓글·활동 로그로 협업하고, 라벨·마감으로 거른다. **데스크톱 우선(1280px+)**, **대담·화려 비주얼**(그라데이션 셸/히어로 + 컬러 아바타 + 강한 lift), 라이트+다크. 실시간 동기화·계정/인증 백엔드는 **이 디자인 범위 밖**(멀티유저는 개념적으로만). **멀티보드/멤버관리/검색/알림 실동작은 셸의 `[범위 밖]` 스텁으로만 표현** — 디자인 표현이지 MVP 실기능이 아니다(00-flow §앱 셸, PRD §11 A-bis).

## 지킬 계약 (이걸 어기면 디자인 의도가 깨진다)
0. **앱 셸 = 두 화면 공유 프레임** — 좌측 사이드바(홈·보드·제품 네비 + 멤버·설정 [범위 밖]) + 상단 탑바(검색·+새로·알림·테마·사용자, 보드엔 역할도). **현재 화면 네비 항목은 링크가 아니라 `aria-current="page"`**(자기 자신으로 가는 죽은 링크 금지). 멤버·설정·마케팅 보드·+새로·검색·알림/사용자 메뉴 액션은 **`[범위 밖]` 스텁**으로만(실기능 만들지 말 것). 셸은 그라데이션 브랜드(`--gradient-brand`), 홈 히어로는 `--gradient-hero` 위 흰 텍스트(AA 통과).
1. **카드 표면 정보밀도 절제** — 표면엔 제목(2줄 말줄임)·담당자 아바타·라벨칩 1~2·마감일(overdue 강조)·댓글수(total)까지만. 설명·전체 댓글·활동은 **상세 패널**로. 더 얹지 말 것.
2. **카드 이동 = 두 경로, 한 결과** — 마우스 네이티브 드래그&드롭 + 키보드/메뉴 경로(카드 "⋮ 이동") 둘 다 제공. 결과는 동일: columnId·정렬 위치 변경 + 양 컬럼 카운터 갱신 + 활동 로그 'moved' 1건. 키보드 경로는 a11y 필수(드래그만 두지 말 것).
3. **드래그 상태를 실제로 렌더** — 드래그 가능 신호(hover lift)·드래그 중(들림/플레이스홀더)·드롭존 하이라이트·빈 컬럼 드롭존. 정적 카드만 그리지 말 것.
4. **통합 타임라인** — 댓글 + 활동을 하나의 시간 역순 스트림으로(전체/댓글/활동 필터). 활동은 사람이 읽는 한국어 문장("OOO 님이 진행 중으로 이동함"). 댓글은 @멘션 칩 전제.
5. **필터는 파생 뷰 상태** — 담당자(멀티)·라벨·마감(오늘/이번주/지남)·"내 카드만". 데이터가 아니라 표시 술어. 컬럼 카운트는 필터된 수를 반영. 0건이면 빈 보드 상태.
6. **역할(편집자/뷰어)** — 뷰어는 순수 읽기전용: 드래그·추가·편집·배정·댓글 입력 전부 차단/숨김 + "읽기 전용" 안내. (개념적 권한 — 실제 인증은 백엔드 몫.)
7. **토큰 2단 + 라이트/다크 페어** — semantic 토큰만 컴포넌트에 노출. 다크는 단순 반전 아니라 별도 페어 스케일 + 4단 elevation. **라이트·다크 양 모드 WCAG AA**(본문 4.5:1, UI/큰 글자 3:1) 유지 — 특히 라벨칩(흰 텍스트)·overdue 칩·촘촘한 그레이 대비.
8. **접근 이름 보존** — role·aria-label·텍스트 식별자를 유지하면 동봉 검증 spec(아래)이 대상 구현에도 그대로 돈다.
9. **데이터 모델** — `../PRD.md` §5의 엔터티(Board·Column·Card·Member·Label·Comment·Activity)와 관계를 따른다. 핵심: Card.position은 **분수 인덱싱**(드래그 재정렬 시 인접 카드 중간값), @멘션은 Comment.body 파싱(별도 참조 없음), 댓글 'resolved' 상태는 **MVP 없음**(카드 표면은 댓글 total만). MVP 담당자 1명(+미배정)이되 모델은 다대다 비파괴.

## 만들 순서 (권장)
1. **토큰 이식** — `token-mapping.md`로 대상 토큰 시스템에 semantic 역할을 매핑(그라데이션 `--gradient-brand/-hero`·`--shadow-bold`·`--ring-accent` 포함). 라이트/다크 페어 + a11y 대비부터 잠근다.
2. **데이터 모델/시드** — PRD §5 엔터티로 타입·시드 목업(컬럼 5·멤버 5·라벨 6·카드 6) 구성. 기준일 고정값으로 overdue/이번주 파생.
3. **앱 셸** — 사이드바 네비(그라데이션 브랜드) + 탑바(검색·알림·테마·사용자, 보드엔 역할). 현재 화면=aria-current, [범위 밖] 스텁 표시. 두 화면이 이 셸을 공유.
4. **홈 대시보드(index)** — 히어로 인사 + 내게 배정된 카드(→보드) + 최근 활동 + 내 보드 타일.
5. **보드 영역** — 헤더(제품 보드·역할·테마는 탑바) + 필터바 + 컬럼 영역(컬럼 5개, 컬럼 영역만 가로 스크롤). 1280/1440 무파손.
6. **카드 + 드래그&드롭** — 카드 표면 + 네이티브 DnD(시각 상태) + 키보드/메뉴 이동. 컬럼 카운터·활동 로그 연동.
7. **상세 패널** — 오버레이 다이얼로그(role=dialog) + 통합 타임라인 + @멘션 댓글 입력 + 담당자 변경.
8. **필터 + 빈/경계 상태** — 필터 술어·카운트 반영·0건 빈 상태·미배정·overdue·아바타 +N·과밀 스크롤·뷰어 읽기전용.
9. **역할/테마 토글** — 뷰어 차단 로직 + 테마 영속 + 양 모드 a11y 회귀.

## 검증 (이식 권장 — 계약 회귀 방지)
동봉 spec은 포터블 셀렉터(role·aria·텍스트)라 **접근 이름만 유지하면 대상 구현에도 그대로 돈다**:
- `../wire.spec.js` — 흐름·기능 완결성(이동·배정·필터·댓글·추가·뷰어·테마 21항목, 죽은 컨트롤 0, 1280/1440 무파손).
- `../tokens.spec.js` — 토큰 대비 AA(라이트/다크).
- `../screens-a11y.spec.js` — 렌더 axe(라이트/다크) serious 0.
대상 스택에 맞춰 셀렉터 그대로 재사용하거나, 동등 테스트로 옮기세요.
