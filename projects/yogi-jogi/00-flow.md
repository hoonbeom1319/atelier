# 00-flow — yogi-jogi (발자취 앱) 화면·플로우 합의본

> 입력: `PRD_발자취앱.md` + `디자인브리프_발자취앱.md`
> 이 문서는 **무엇을 그릴지**의 합의본이다. 색·픽셀이 아니라 **화면 목록 / 이동(분기) / 상태**를 못박는다.
> 한 줄 주문: *흰 바탕에 사진·지도가 주연, 따뜻한 코랄 포인트. UI는 미니멀, 검토·보정 화면을 가장 매끄럽게.*

---

## 0. 앱 한 장 요약

다녀온 곳 사진 무더기를 던지면 → EXIF로 동선·방문지(Stop)를 **자동 조립** → 미분류 사진만 톡톡 **보정** → 짧은 글·태그 얹어 **코스 완성** → 코스가 쌓여 **발자취 지도**가 된다. 2층(얇은 연결)으로 일방 팔로우 기반 "다 같이" 피드.

- **플랫폼:** 모바일 웹 우선 (뷰포트 390px 기준). 데스크톱 후순위.
- **하단 탭 3 + 가운데 플로팅:** ① 발자취 지도(홈) · ② 다 같이 · ③ 프로필 / 가운데 **＋ 새 코스**(코랄, 최강조).
- **기본 비공개.** 공유는 코스별 명시적 공개 시에만.

## 2. 사용자 플로우 (A→B, 분기)

### 핵심 흐름 (이게 막히면 앱이 죽음)
```
[로그인 S12] → (코스 0개) [빈 상태 S1] ──"첫 코스 만들기"──▶ [사진 선택 S2]
                                                              │ 다중 선택 후 "조립"
                                                              ▼
                                                    [자동 조립 로딩 S3]
                                                              │ 완료
                                                              ▼
                                              ┌──── [검토·보정 S4] ★ ────┐
                                              │  (미분류 보정 인터랙션)   │
                                              │   ↺ 시트/검색/합치기      │
                                              └──────────"다음"──────────┘
                                                              ▼
                                                       [마무리 S5]
                                                  (제목·글·태그·공개여부)
                                                              │ "완성"
                                                              ▼
                                                     [코스 상세 S6]
                                                              │ 합산
                                                              ▼
                                              [발자취 누적 지도 S7] 핀 +1
```

### 탭 네비게이션 (전역)
```
하단탭:  [발자취 지도 S7] ── [＋ 새 코스 → S2] ── [다 같이 S9] ── [프로필 S10]
           │ 코스 0개면 S1으로 대체
           └ 지도 ↔ 리스트(S8) 토글
```

### 분기 / 곁가지
- **S4 검토·보정 내부 분기 (★승부처):**
  - 미분류 사진 **탭 선택**(복수 가능) → 하단 **Stop 시트** 올라옴 → 근처 Stop 추천 탭 → 끝
  - 위치 자체 없는 사진 → **지역명 검색** → 결과 탭 / 실패 시 **핀 직접 이동** 모드
  - Stop **합치기 / 쪼개기** (카드 길게 누르기)
- **S9 다같이 → S10 프로필:** 카드의 작성자 탭 → 프로필 → "팔로우"(일방, 즉시)
- **S6 코스 상세 → 공유:** "공유" → 링크 생성 → (앱 없는 수신자) **S11 웹뷰**
- **뒤로/취소:** S2~S5는 상단 ← 로 이전 단계 복귀. S4/S5에서 나가면 "임시저장?" 확인(막다른 길 방지).

## 6. 열린 질문 (디자인에 영향 주는 것만)

1. **홈(S7) 기본 뷰:** 지도 우선 + 리스트(S8) 토글로 가정. 리스트를 별도 탭이 아니라 토글로 둬도 OK?
2. **검토·보정(S4) 지도 비중:** 상단 지도는 "접고 펼 수 있게"가 기본 — 처음 진입 시 **펼침**(동선 먼저 보여줌)으로 가정. 맞나?
3. **샘플 코스(S1):** 가족 주말 나들이 예시 1개를 더미 데이터로 그린다(전주 한옥마을류). 지역/내용 지정 있나, 아니면 제가 자연스럽게?
4. **로그인(S12) 위치:** MVP 와이어에 포함하되 묶음 1 끝자락에 최소로. 흐름 테스트는 S1부터 시작해도 되나(로그인 스킵)?

> 위 4개는 가정값으로 진행 가능 — 다르면 알려주시고, 없으면 그대로 갑니다.
