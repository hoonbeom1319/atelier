## §0. 대상 플랫폼 / 뷰포트 / 톤

- **플랫폼:** 모바일 웹 (추후 웹뷰로 앱화 — 현 디자인에 영향 없음).
- **뷰포트(확정):** **모바일 우선 단일 컬럼.** 콘텐츠 컬럼은 폭 ~390px, **최소 360px에서 안 깨지게** 유동.
  데스크톱에선 페이지가 깨지지 않고 넉넉히 보이도록, 앱 컬럼을 **중앙 정렬한 반응형 셸**로 감싼다(페이지 **max-width 1280px**, 더 큰 1920px 화면에선 여백 배경). 즉 *앱 자체는 폰 폭, 바깥은 데스크톱에서도 단정하게*.
  - 구현: `body`는 옅은 배경, 앱은 `.app{ width:min(100%,420px); min-width:360px; margin:0 auto; }` 식 중앙 폰 컬럼. 하단 탭·플로팅＋는 이 컬럼 기준으로 고정.
  - **단, 반응형 셸(min/max-width·데스크톱 여백)은 hi-fi(5단계)에서 입힌다.** 와이어(2단계)는 **단순 폰 폭 단일 컬럼**으로만 — 와이어 목적은 흐름·상태이지 폭 적응이 아니므로.
- **톤:** 따뜻하고 가벼운 기록 도구. UI(크롬)는 미니멀하게 빠지고 콘텐츠(사진·지도)는 인스타처럼 크고 시원하게.
- **포인트 컬러:** 코랄 `#FF6B5C` 단 하나(주요 액션·선택 상태·지도 동선·활성 탭). 바탕은 웜화이트 `#FAFAF8`, 텍스트 `#2B2B2B`.
- **다크모드:** 범위 밖(라이트만).

### ★ 지도 표현 규약 (외부 의존성 0 제약과 직결 — 매우 중요)
산출물은 외부 라이브러리 없는 독립 HTML이다. 따라서 **실제 카카오맵을 렌더하지 않는다.**
지도는 **CSS/SVG로 그린 양식화된 faux-map**으로 표현한다:
- 옅은 회색 지형/도로 느낌 배경(SVG 또는 CSS 그라데이션·격자).
- **동선 = 코랄(`#FF6B5C`) polyline**(SVG `<path>` 또는 `<polyline>`).
- **Stop = 핀 마커**(번호 배지). 누적 지도는 핀이 여러 개.
- 와이어 단계: 회색 박스에 "지도 영역" 라벨 + 동선 선 형태만. hi-fi: 양식화·옅은 톤.
- 지도 *조작*(핀 직접 이동 등)은 faux 좌표 위에서 상태 변화로만 표현(실제 타일맵 없음).

## §2. 사용자 플로우 (A→B · 분기)

### 핵심 흐름 (이게 막히면 앱이 죽음)
```
C1 빈 상태 ──[첫 코스 만들기]──▶ C2 사진 선택 ──[N장 선택·완료]──▶ C3 자동 조립 로딩
   └─(코스 있으면 S1 홈에서 ＋새 코스로도 진입)
C3 ──(자동 완료)──▶ C4 검토·보정 ★
C4 ──[다음/완료]──▶ C5 마무리 ──[코스 완성]──▶ C6 코스 상세
C6 ──(발자취에 합산)──▶ S1 홈 누적 지도
```

### 검토·보정(C4) 내부 분기 — 미분류 사진 N장 처리의 **모든 경로가 완료에 도달해야** 함
```
미분류 사진 탭(다중 선택) → [Stop 지정 시트] ┬─ ① 근처 Stop 추천 탭 → 지정 완료
                                            ├─ ② 기존 Stop 목록서 탭 → 지정 완료
                                            └─ ③ 새 Stop 추가 → 지정 완료
위치 자체 없는 사진 → [지역명 검색] ┬─ 결과 탭 → Stop 생성/지정
                                  └─ (검색 실패) → 핀 직접 이동 모드 → 확정
Stop 카드 길게 누르기 → 합치기 / 쪼개기
모든 경로의 종착 = 미분류 수 0 또는 사용자가 남긴 채 [다음]
```

### 연결(2층) 흐름
```
S2 다 같이 ──[코스 카드 "코스 보기"]──▶ C6 코스 상세(읽기)
S2 다 같이 ──[이름·아바타 탭]────────▶ S3b 다른 사람 프로필 ──[팔로우 토글]──▶ 팔로잉
S2 다 같이 ──[헤더 검색 / 빈상태 '사람 찾기']─▶ S7 사람 찾기
S7 사람 찾기 ──[검색→결과 팔로우]──▶ 팔로잉 / [결과 탭]──▶ S3b 다른 사람 프로필
탭바 '프로필' ──▶ S3 내 프로필 ──[⚙설정]──▶ S6 설정 ──[로그아웃]──▶ S5 로그인
C6 코스 상세 ──[공유]──▶ 링크 생성 → (앱 없는 사람) S4 웹뷰
S5 로그인 ──[3종 중 택1]──▶ (최초) C1 빈 상태 / (기존) S1 홈
```

## §3. 화면별 구성요소 · 상태 (빈/로딩/정상/에러) — spec 파생 원천

### C4 검토·보정 ★ (역량 집중, 가장 정교)
- **상단:** faux-map(코랄 동선 + 번호 Stop 핀). **접기/펼치기** 가능.
- **중단:** Stop 카드 **세로 리스트**(가로 스크롤 X). 카드 = 장소명 + 도착시각 + 사진 썸네일 N장.
- **하단:** **"미분류" 묶음** — 위치 못 정한 사진 모음(개수 표시).
- **상태:** 로딩(조립 중 진행률) / 정상·미분류 있음 / 정상·미분류 0(완료 느낌) / (엣지) 전부 미분류(EXIF 없음).
- **책임 동작(2층 검증 대상):** 미분류 사진 탭 선택(다중·카운터) → 시트 → Stop 지정으로 미분류 −N / 추천·기존·신규 3경로 모두 완료 도달 / 지역검색→핀 경로 동작 / 합치기·쪼개기 / 지도 접기·펼치기.

### C1 빈 상태
- 한 줄 설명 + **샘플 코스 1개**(완성 예시 카드, faux-map 썸네일) + 큼직한 코랄 **"첫 코스 만들기"**. 긴 튜토리얼 금지.
- 상태: 빈 상태 그 자체(이 화면이 곧 코스 0개 상태).
- 책임 동작: "첫 코스 만들기" → C2 이동 / 샘플 코스 탭 → C6 미리보기.

### C2 사진 선택
- 갤러리 **다중 선택 그리드** + 선택 개수 표시 + [완료].
- 상태: 0장 선택(완료 비활성) / N장 선택(카운터·완료 활성).
- 책임 동작: 썸네일 탭 → 선택 토글(체크·카운터 ±1) / [완료] → C3.

### C3 자동 조립 로딩
- "사진에서 장소를 찾는 중…" + 진행 상태(멈춘 느낌 금지). 큰 점프라 별도 화면으로 모델링.
- 상태: 로딩(진행률) → 자동 전이 C4.

### C5 마무리
- 코스 제목 입력 / 짧은 글·태그(선택·비워도 완성) / **공개여부 토글(기본 비공개)**.
- 상태: 정상. 공개 토글 ON 시 "공유됨" 안내 노출.
- 책임 동작: 제목 입력 / 태그 추가·삭제 / 공개 토글 뒤집힘(`aria-pressed`) / [코스 완성] → C6.

### C6 코스 상세
- 상단 풀블리드 faux-map(코랄 동선). 아래 사진 인스타처럼 크게(Stop 순). 제목·날짜·글·태그. **공유 버튼**.
- 상태: 정상 / (자기 코스) 공개여부 표시 / (남의 공개 코스) 읽기.
- 책임 동작: 공유 → 링크 생성(상태·토스트) / 사진 스크롤.

### S1 발자취 누적 지도(홈)
- 모든 코스 Stop을 한 faux-map에(핀 누적). 내 코스 목록 접근. 하단 탭 셸.
- 상태: **코스 0개 → C1로 대체** / 정상(핀 누적).
- 책임 동작: ＋새 코스 → C2 / 코스 핀·목록 탭 → C6.

### S2 다 같이
- 팔로우한 사람 코스 **시간순 카드 목록** + **안 읽음 배지**(빨간 점/개수). **헤더에 '사람 찾기' 진입(검색 아이콘).**
- 카드 구조: **이름·아바타 = 그 사람 프로필(S3b) 진입 / "코스 보기" = 코스 상세(C6)** (둘은 분리된 타깃).
- 상태: **빈(팔로우 0명)** — "친구를 팔로우하면…" + **사람 찾기(S7) 진입점**(반드시) / 정상(목록·배지) / 다 읽음(배지 0).
- 책임 동작: 안 읽음 표시 → 배지 −1 / 이름·아바타 → S3b / 코스 보기 → C6 / 사람 찾기 → S7.

### S3 내 프로필 (탭 루트)
- **내 코스 그리드**(사진 중심) + **⚙설정 진입**. **팔로우 버튼 없음(나 자신).**
- 상태: 정상(내 코스). 코스 0개면 빈 상태 안내(간단).
- 책임 동작: 코스 그리드 탭 → C6 / ⚙설정 → S6.

### S3b 다른 사람 프로필
- 그 사람 **공개 코스 그리드** + **일방 팔로우 버튼**(팔로우↔팔로잉). 뒤로가기.
- 진입: **다같이 이름·아바타** 또는 **사람 찾기(S7) 결과**. (탭바에서 직접 못 옴 — 컨텍스트 전용.)
- 책임 동작: 팔로우 토글(라벨·`aria-pressed` 변화) / 공개 코스 탭 → C6 / 뒤로.

### S7 사람 찾기
- **닉네임 검색 입력** → 결과 리스트(각 결과 = 이름 + **팔로우 버튼** + 탭하면 S3b 프로필).
- 진입: 다같이 헤더 검색 / 다같이 빈 상태 '사람 찾기'.
- 상태: 초기(검색 전 안내) / 결과 있음 / (결과 없음 안내).
- 책임 동작: 검색 입력 → 결과 노출 / 결과 팔로우 토글 / 결과 탭 → S3b.

### S4 코스 웹뷰
- 앱 없는 사람용 읽기 전용 코스(faux-map + 사진 + 글). "앱에서 열기/만들기" CTA.
- 상태: 정상(공개 코스) / (비공개 링크) 접근 불가 안내.

### S5 로그인
- 카카오(가장 크게) / 구글 / 이메일 3종.
- 상태: 정상 / (이메일) 입력 폼.
- 책임 동작: 3종 버튼 → (데모) 진입 처리.

### S6 설정
- 최소: 로그아웃, 계정. 
- 책임 동작: 로그아웃 → S5.

## §4. 인터랙션 모델 (모양 아닌 **구조** — 화면 골격을 좌우)

| 모델 | 화면 | 구조적 함의 |
|---|---|---|
| **다중 선택 + 카운터 + 시트 액션** | C2 사진선택, C4 미분류 보정 | 선택 모드·카운터·하단 시트가 화면 구조에 들어감 |
| **탭-투-지정(드래그 금지)** | C4 | 사진 탭 → Stop 목록 시트 탭. 근처 추천을 시트 최상단에 |
| **검색 → 결과 → (실패)핀 이동** | C4 위치없는 사진 | 검색창·결과 리스트·핀 이동 모드 3상태 |
| **롱프레스 → 합치기/쪼개기** | C4 Stop 카드 | 카드 컨텍스트 액션 |
| **단일 토글** | C5 공개여부, S3b·S7 팔로우, C4 지도 접기 | `aria-pressed`로 상태 표현 |
| **배지 카운터(읽음 소거)** | S2 다같이 | 안 읽음 표시 → −1 |
| **이름·아바타 = 사람 진입** | S2 다같이, S7 결과 | 사람 식별 영역 탭 → S3b 프로필 (코스 영역 탭과 분리) |
| **검색 → 결과 리스트** | S7 사람 찾기 | 검색 입력 → 결과 노출 → 각 결과 팔로우/프로필 |
| **탭 셸 + 플로팅 ＋** | 전역 | 하단 탭 3(발자취·다같이·**내 프로필**) + 가운데 코랄 ＋ |

> **드래그는 명시적으로 금지**(PRD·브리프). 탭-투-지정으로만.

## §6. 묶음(청크) 진행 계획 — 큰 PRD

단계(세로)로 자르고, 단계 안에선 묶음마다 게이트. `tokens.css`는 전 묶음 구조 확정 후 1회.

- **묶음1 — 기록 루프(심장):** C1 빈상태 · C2 사진선택 · C3 로딩 · **C4 검토·보정★** · C5 마무리 · C6 코스상세 (6)
- **묶음2 — 발자취 & 연결:** S1 홈 누적지도 · S2 다같이 · S4 웹뷰 (3)
- **묶음3 — 계정·주변:** S3 프로필 · S5 로그인 · S6 설정 (3)

```
와이어: 묶음1 검증·OK → 묶음2 검증·OK → 묶음3 검증·OK
  → 전체 도달성 크롤 1회 → tokens.css(1회) → 하이파이(묶음별) → 인터랙션 검증 → handoff
```
묶음 단독 검증 시 다음 묶음 링크는 스텁/`[범위 밖(다음 묶음)]`으로 막다른 길 방지.
