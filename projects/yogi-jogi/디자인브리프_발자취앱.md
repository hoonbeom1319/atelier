# 디자인 브리프 — (가칭) 발자취 앱

> 이 문서는 **클로드디자인 입력 전용**이다. 기능·근거는 PRD를 참조하고, 여기서는 **화면을 그리는 데 필요한 시각적 결정**만 담는다.
> 핵심 원칙 한 줄: **UI(크롬)는 미니멀하게 빠지고, 콘텐츠(사진·지도)는 인스타처럼 크고 시원하게.**

---

## 0. 절대 원칙 (모든 화면에 적용)

1. **콘텐츠가 주연, UI는 조연.** 사진과 지도가 화면의 주인공. 버튼·메뉴·텍스트는 조용히 비킨다.
2. **사진은 풀블리드(가장자리까지 꽉 차게), UI는 여백 넉넉히.** 미니멀과 인스타 감성을 "역할 분리"로 양립시킨다.
3. **색은 한 곳에만.** 바탕은 무채색, 포인트 컬러는 핵심 액션과 지도 동선에만.
4. **모바일 우선.** 한 손 엄지 조작 기준. 핵심 액션은 화면 하단에.
5. **검토·보정 화면에 디자인 역량을 몰아라.** 이 앱의 승부처. 다른 화면이 평범해도 이 화면은 매끄러워야 한다.

---

## 1. 톤 & 비주얼 방향

- **무드:** 깔끔·미니멀하되 차갑지 않게. 따뜻하고 가벼운 기록 도구. 30대 가족 정서.
- **UI 스타일:** 토스·노션처럼 절제된 크롬(여백, 얇은 구분선, 최소한의 그림자). 단 콘텐츠 영역은 인스타·핀터레스트처럼 사진이 시원하게.
- **모서리:** 부드러운 라운드(카드·버튼 12~16px 정도). 각지지 않게 — 따뜻함 표현.
- **그림자:** 아주 옅게만. 떠 있는 느낌보다 평평·깔끔하게.

### 참고(레퍼런스) — 무엇을 가져오고 무엇을 버릴지 명시
| 레퍼런스 | 가져올 것 | 버릴 것 |
|---|---|---|
| 런닝 앱(Strava류) | **지도 위 동선 시각화**(경로를 선으로 그리는 방식) | 어둡고 스포티한 톤, 경쟁/숫자 강조 → **정반대로 부드럽게** |
| 인스타그램 | 사진을 크고 시원하게 보여주는 콘텐츠 우선 레이아웃 | 복잡한 소셜 기능 UI(좋아요·댓글·스토리 바 등) → MVP엔 없음 |
| 핀터레스트 / 데이원 | 깔끔한 카드, 사진 중심, 차분한 크롬 | 무한 탐색·발견 피드 → 우리는 탐색 앱 아님 |

---

## 2. 컬러 (코드값 포함 — 이대로 적용)

> 색 논리: 사진이 주연이라 UI 색이 화려하면 사진과 싸운다. 그래서 **바탕은 무채색, 포인트는 단 하나.**

| 역할 | 색 | 코드(참고값) | 쓰는 곳 |
|---|---|---|---|
| **메인 / 포인트** | 따뜻한 코랄 | `#FF6B5C` | 주요 버튼, 선택된 상태, **지도 동선 선**, 활성 탭 |
| 메인 진하게 | 딥 코랄 | `#E8503F` | 눌림 상태, 강조 텍스트 |
| 텍스트 / 아이콘 | 차분한 다크그레이 | `#2B2B2B` | 본문·아이콘 (완전 검정 금지 — 쌀쌀함) |
| 보조 텍스트 | 미디엄 그레이 | `#8A8A82` | 날짜, 캡션, 비활성 |
| 배경 | 웜 화이트 | `#FAFAF8` | 화면 바탕 (순백보다 살짝 따뜻하게) |
| 카드/구분선 | 옅은 웜 그레이 | `#EDEBE6` | 카드 배경, 얇은 라인 |

- **동선 선은 반드시 코랄(`#FF6B5C`).** 옅은 지도 위에서 또렷하게 보이고 "내 발자취"의 상징색.
- 대안(사용자 변심 시): 세이지 그린 `#5B8C6E`(자연·여행), 딥 코발트 `#3A5BD9`(또렷·모던). **1순위는 코랄.**

---

## 3. 타이포 & 아이콘

- **폰트:** 한글 가독성 좋은 산세리프(예: Pretendard 계열). 둥근 기운이 살짝 있는 것이 톤에 맞음.
- **위계:** 코스 제목 > 장소명 > 날짜·캡션 순으로 명확히. 화면당 강조는 하나만.
- **아이콘:** 얇은 선(line) 아이콘. 꽉 찬 아이콘은 활성 상태에만.

---

## 4. 전체 구조

- **하단 탭 3개 + 가운데 플로팅 액션:**
  - 탭1: **발자취 지도**(메인/홈) · 탭2: **다 같이** · 탭3: **프로필**
  - 가운데: **＋ 새 코스** (코랄, 가장 눈에 띄게 — 핵심 액션이므로)
- **메인(첫 화면) = 발자취 지도.** 단, **코스 0개면 빈 상태 화면으로 대체**(아래 5장).

---

## 5. 화면별 구성 요소 + 상태

> 각 화면은 **빈 상태 / 로딩 / 정상**을 가질 수 있다. 빈 상태를 반드시 함께 디자인할 것(신규 사용자의 첫인상).

### ⭐ 5-1. 검토·보정 화면 (최우선 — 역량 집중)
이 앱의 승부처. 사진을 던진 직후 자동 조립 결과를 확인·보정하는 화면.
- **상단:** 그날 동선이 그려진 지도(코랄 선 + Stop 핀). 접고 펼 수 있게.
- **중단:** Stop(방문지) 카드들이 **세로 리스트**(가로 스크롤 X — 장소 많아도 안 밀리게). 각 카드 = 장소명 + 도착시각 + 그 안 사진 썸네일 몇 장.
- **하단:** **"미분류" 묶음** — 위치 못 정한 사진들 모아둠.
- **핵심 인터랙션(드래그 금지):**
  - 미분류 사진 **탭 선택**(여러 장 가능) → 하단에서 **Stop 목록 시트가 올라옴** → 넣을 Stop 탭 → 끝.
  - 시트 상단에 **"근처 Stop 추천"** 1~3개 먼저(한 번 탭으로 끝나게).
  - 위치 자체 없는 사진: **지역명 검색창** → 결과 탭, 실패 시 **핀 직접 이동** 모드.
  - Stop **합치기/쪼개기** 진입점(카드 길게 누르기 등).
- **상태:** 로딩(자동 조립 중 진행률) / 정상(미분류 있음) / 정상(미분류 0 — 깔끔한 완료 느낌).
- **느낌:** 보정이 "숙제"가 아니라 "톡톡 정리하니 완성되는" 가벼운 경험이어야. 마찰 최소.

### 5-2. 빈 상태 (코스 0개) — 첫인상
- 한 줄 설명: "다녀온 곳 사진을 올리면 지도에 코스가 그려져요"
- **완성 예시(샘플 코스) 1개**를 보여줌(글 설명보다 강력) — 가족 나들이 예시.
- 큼직한 코랄 **"첫 코스 만들기"** 버튼. (긴 튜토리얼·슬라이드 금지)

### 5-3. 사진 선택 → 자동 조립 로딩
- 갤러리 다중 선택. 선택 개수 표시.
- 로딩: "사진에서 장소를 찾는 중…" + 진행 상태(수십~수백 장이라 시간 걸림, 멈춘 느낌 금지).

### 5-4. 마무리
- 코스 제목 입력, 짧은 글·태그(선택 — 비워도 완성).
- **공개여부 토글(기본: 비공개).** 공개로 켤 때만 공유됨을 분명히.

### 5-5. 코스 상세
- **상단: 동선 지도(코랄 선)** 풀블리드로 시원하게.
- 아래: 사진들이 인스타처럼 크게(Stop 순서대로). 제목·날짜·글·태그.
- 공유 버튼(링크 생성).

### 5-6. 발자취 누적 지도 (메인/홈)
- 내 모든 코스의 Stop을 한 지도에. 많이 다닐수록 핀이 쌓여 "발자취"가 보이는 쾌감.
- **코스 0개면 5-2(빈 상태)로 대체.**

### 5-7. 다 같이
- 팔로우한 사람들의 코스 시간순 카드 목록 + **안 읽음 배지**(빨간 점/개수).
- **빈 상태(팔로우 0명):** "친구를 팔로우하면 여기 뜨고, 누가 새로 올리면 알려줘요" + 사람 찾기/팔로우 진입점. ← 반드시 디자인.

### 5-8. 프로필 / 로그인·설정
- 프로필: 내 코스 그리드(사진 중심), 일방 팔로우 버튼.
- 로그인: 카카오 / 구글 / 이메일 3종. 카카오를 가장 크게(타깃 마찰 최소).
- 설정: 최소한(로그아웃, 계정).

## 6. 디자인에서 의도적으로 빼는 것 (그리지 말 것)
- 좋아요·댓글·푸시 알림 UI (Later)
- 친구 요청/수락/차단 UI (일방 팔로우라 불필요)
- 코스 탐색·추천·검색 피드 (탐색 앱 아님)
- 다크모드 (Later — 라이트 기준으로만)
- Stop별 글·태그 입력칸 (글·태그는 코스 단위만)

---

## 7. 우선순위 (디자인 역량 배분)
1. **검토·보정 화면** (★ 최우선, 가장 공들일 것)
2. 코스 상세 / 발자취 지도 (앱의 얼굴)
3. 빈 상태들(첫인상)
4. 다 같이 / 마무리
5. 로그인·설정·프로필 (기능적이면 충분)

---

## 8. 한 줄 요약 (도구에 거는 핵심 주문)
"흰 바탕에 사진과 지도가 주인공인, 따뜻한 코랄 포인트의 모바일 기록 앱. UI는 미니멀하게 빠지고, 검토·보정 화면을 가장 매끄럽게."
