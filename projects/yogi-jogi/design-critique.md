## 빌드 전 비주얼 레퍼런스 브리프

요기조기 hi-fi 빌드 비주얼 브리프 (인스타식 모노크롬 UI + 컬러 사진 / 모바일 390×844 라이트 한국어). 방향을 바꾸지 말 것 — 트렌드 추종 금지. 목표는 "이 모노크롬 방향 안에서 어떻게 와이어가 아닌 진짜 hi-fi가 되나". 모노크롬이라 색으로 숨을 데가 없으니 깊이는 오직 (1)타이포 위계 대비(fw 700/600 헤딩 vs 400 본문, fs 21/17/15/13/11 리듬 실제 사용) (2)여백 리듬(4px 베이스 스케일을 실제로 다르게 — 카드 안 12/16, 섹션 간 24) (3)헤어라인 보더 1px #dbdbdb (4)절제된 그림자(플로팅 요소에만)에서 나와야 한다. 두 개의 치명적 와이어 함정을 반드시 피한다: ①사진은 회색 박스 금지 — 반드시 풍부한 컬러 그라데이션 실사 장면(따뜻한 카페 톤·초록 공원·도시 야경). ②지도 히어로는 밋밋한 회색 사각형 금지 — 도로망 라인·블록·공원/수역 톤차·라벨이 있는 '진짜 지도처럼' 보이는 텍스처 위에 검정 동선 폴리라인+검정 티어드롭 핀을 얹는다. feed 홈은 '지도가 주인공, 사진은 조연'인 지도 히어로 카드가 핵심 차별점이니 여기에 밀도를 몰아준다. 임계 기준: 렌더 스크린샷에서 지도 캔버스와 사진 썸네일이 각각 '텍스처 있는 실제 지도'와 '컬러 실사'로 읽혀야 통과.

### 적용 패턴(현행 기준)

- 타이포 위계 실사용: display 26/h1 21/h2 17/body 15/sm 13/xs 11을 화면마다 실제로 3단계 이상 대비되게 쓴다. 인스타식 = username fw600 13~15px, 캡션 본문 fw400 15px(username 프리픽스만 굵게), 타임스탬프 fw400 11px muted 대문자 느낌. 헤딩과 본문이 같은 weight/size면 즉시 와이어로 읽힘.
- 여백 리듬(밀도): 카드 내부 패딩 12~16px, 카드 사이 divider(1px #f2f2f2) 또는 12~16px 갭, 섹션 헤더 위 24px. 인스타는 콘텐츠를 거의 풀블리드로 밀고 크롬만 얇게 — 좌우 거터를 과하게 주지 말고 미디어는 카드 폭을 꽉 채운다. 균일한 8px 간격 반복(와이어 특징) 금지 — 위계에 따라 간격을 다르게.
- 지도 히어로 캔버스(feed 카드의 주인공): 밋밋한 #ebebeb 사각형 금지. 실제 지도 텍스처를 CSS/SVG로 그린다 — 베이스 #ebebeb 위에 도로망을 흰색(#ffffff)·연회색(#f5f5f5) 라인으로 여러 방향 교차, 블록(건물)을 아주 옅은 회색 폴리곤, 공원/수역은 살짝 다른 톤(공원=옅은 회녹 #e6ebe6, 수역=옅은 회청 #e6e9ed 정도의 절제된 톤차 허용 — 무지개 아님). 그 위에 검정 동선 폴리라인(2.5~3px, stroke-linejoin round, 살짝 투명도 없이 또렷), 검정 티어드롭 핀(흰 아이콘/번호, 작은 drop-shadow). 종횡비 약 4:3~1:1로 카드 상단을 크게 차지.
- 컬러 사진 플레이스홀더: linear-gradient 다중 스톱으로 실제 장면 뉘앙스를 낸다(예 카페=#e8c9a0→#c98a5a 따뜻한 우드톤, 공원=#a8c98a→#5a8a4a 초록, 야경=#3a4a6a→#c98a5a 남보라→주황). 단색·회색 절대 금지. 썸네일은 rounded 8px, object-fit cover 느낌. 가능하면 그라데이션+미세 노이즈/오버레이로 '사진 같은' 깊이.
- feed 지도 히어로 카드 앤아토미: [헤더 56px: 32px 원형 아바타(컬러 그라데) + username fw600 + 동네·시간 muted] → [지도 히어로 대형] → [작은 컬러 사진 썸네일 가로 스크롤 행 72~80px] → [액션 행: 좋아요 하트(레드 #ed4956 채움 토글)+개수 fw600, POI 칩(검정 테두리 pill), 저장] → [메모 fw400 + 태그 칩]. 카드 간 헤어라인 divider.
- 상태·마이크로 인터랙션(죽은 컨트롤 0): 좋아요 하트 아웃라인↔레드 채움 토글+개수 증가, 사진 썸네일 가로 스크롤/캐러셀 도트, 팔로우 토글, 세그먼트 활성(비인터랙티브 span), 바텀시트 열림+scrim(rgba(0,0,0,.55)). 탭 활성은 aria-current + 검정 아이콘(비활성은 gray-500). 전이는 인스타식으로 절제(트랜스폼/오퍼시티 짧게).
- 레드 액센트 규율: 색은 좋아요 하트·미열람 닷(#ed4956)에만. 그 외 모든 액션/핀/칩/버튼/탭 활성은 검정(ink-900). FAB '+'는 검정. 이 절제가 지켜져야 컬러 사진이 살고 인스타 감성이 난다 — 액센트를 여기저기 흩뿌리면 방향 붕괴.
- elevation/깊이: 인스타는 거의 플랫 — 그림자는 앱바(스크롤 시 하단 헤어라인), 바텀시트(shadow-3), FAB 정도만. 카드 본문은 그림자 대신 헤어라인 보더로 분리. 과한 카드 그림자(머티리얼식)는 인스타 톤 아님.
- 지도/핀 레퍼런스 일반화: 특정 SDK 마커 스타일에 묶지 말고 카카오맵식 클러스터 버블(검정 원+개수)·티어드롭 핀·동선 폴리라인을 일반 표현으로. 줌아웃 다수 핀은 클러스터로(개별 나열 금지 — PRD 성능 함의).

### 레퍼런스

- Instagram (모노크롬 UI 골격 원본): 흰 배경·검정 액션·헤어라인 divider·풀블리드 미디어·굵은 username/얇은 캡션 위계·슬림 탭바·좋아요 레드 하트. 전반 크롬·소셜 카드·3열 그리드·업로드 흐름의 기준.
- Polarsteps 2025 리프레시 (지도-퍼스트 + 큰 사진): map-first 인터페이스에 큰 사진 디스플레이·슬리커 레이아웃. '지도가 주인공, 사진이 여정을 채운다'는 요기조기 피드 카드 컨셉의 직접 참조. 동선 라인 시각화.
- 카카오맵 (지도 캔버스·핀·클러스터·바텀시트 인터랙션): 단 핀·동선은 모노크롬으로 번역. 장소 검색 결과 리스트(장소명·카테고리·도로명·거리) 앤아토미.
- Google/Apple Photos (upload-2 자동분류 UX): 장소/그룹 카드·추정 vs 확정 배지·in-place 배정 흐름.
- Mapbox Static Images 스타일 정적 지도 썸네일 (feed 카드 지도 히어로): 폴리라인+오리진/데스티네이션 마커를 얹은 정적 지도 표현 = 스크롤 피드에 지도 썸네일을 얹는 표준 패턴의 시각 목표치.

### 안티패턴(즉시 와이어/off-brief 판정)

- 회색 사진 플레이스홀더(단색 #ccc 박스·회색 그라데) — 이 brief의 최우선 금지. 회색 사진은 즉시 와이어프레임으로 읽힌다. 반드시 컬러 실사 그라데 장면.
- 밋밋한 회색 지도 사각형(텍스처 0, 도로·블록·라벨 없이 #ebebeb 한 면) — 두 번째 치명 함정. '지도 히어로가 주인공'인데 지도가 빈 회색 박스면 화면 전체가 와이어. 도로망·블록·수역/공원 톤차·핀·동선으로 지도임이 픽셀에서 보여야.
- 타이포 위계 붕괴: username·캡션·시간이 전부 같은 15px fw400 — 굵기/크기 대비가 없으면 모노크롬 특성상 즉시 와이어. 헤딩·본문·메타를 실제로 다른 weight/size로.
- 균일한 8px 간격을 모든 곳에 반복(레이아웃 리듬 없음) — 와이어의 전형. 위계 따라 간격 차등.
- 레드(또는 임의 색)를 액센트 규율 밖에 흩뿌리기 — 버튼·핀·칩·배지에 색을 넣으면 인스타 모노크롬 방향 배신. 색은 좋아요·미열람 닷에만, 그 외 검정.
- 무지개 카테고리 핀(1차 지도앱 톤으로 회귀) — 이미 폐기된 방향. 핀은 전부 검정, 카테고리는 라벨/아이콘으로 구분.
- 머티리얼식 무거운 카드 그림자·과한 elevation — 인스타는 거의 플랫. 헤어라인 보더로 분리하고 그림자는 시트/앱바/FAB에만.
- 죽은 컨트롤(좋아요·팔로우·캐러셀·세그먼트가 눌러도 상태 불변) — deadControl/stateInert. 전부 와이어드+상태 변화.
- 지도 개별 핀 수백개 나열(클러스터 없이) — PRD 성능 함의 위반, 화면도 지저분.
- off-brief: 트렌드(글래스모피즘·비비드 컬러·다크모드)를 '요즘 유행'이라며 끌어와 모노크롬 방향을 갈아치우기 — 감정 실패. 다크모드는 P1, MVP는 라이트만.

## 빌드 후 트렌드 감정 (렌더 스크린샷 기반)

### 종합 점수: 85 / 임계값 80 (통과, 리디자인 미실행)

방향(인스타식 모노크롬 UI + 컬러 사진) 안에서 완성도가 높은, 명백히 hi-fi 수준의 산출물이다. '와이어 + 색' 함정을 확실히 벗어났다: (1) UI 크롬이 진짜 모노크롬(흰 배경·검정 액션/핀·그레이 위계·헤어라인 보더·절제된 그림자)으로 일관되고 토큰 규율(레드는 좋아요/알림만)을 픽셀에서 지킴, (2) 사진 플레이스홀더가 회색 박스가 아니라 따뜻한 카페 톤·초록 공원·도시 야경 등 풍부한 컬러 그라데이션 — 브리프의 반(反)와이어 요구를 정확히 이행, (3) 컴포넌트 일관성(번호 티어드롭 핀·컬러 아바타+액션 배지·칩·세그먼트·토글 스위치)이 14화면 전반에 유지, (4) 상태 표현이 살아있음(좋아요 채움/빈 하트, 팔로우 채움/팔로잉 아웃라인, 확정/추정 배지 대비, 라디오/스위치 온·오프, 데모 빈상태 토글). 위계·여백 리듬·타이포 스케일이 인스타 문법과 정합하고, feed의 '지도 히어로 카드', post-detail의 동선+캐러셀, upload-4-meta의 게시 폼은 현행 소셜/지도 앱 기준(카드형·둥근 모서리·서브틀 elevation, Polarsteps식 라인+핀, 카드 스캔성)에 부합한다.

임계값(80)은 화면별·종합 모두 통과 — 자동 리디자인은 불필요하다. 다만 천장을 누르는 세 가지 한계가 있다: ① 사진 충실도 — 플레이스홀더가 전부 '방향성 그라데이션'이라 실사 디테일(피사체·질감)이 없어, 사진이 큰 면적을 차지하는 화면(post-detail 캐러셀·map-home 그리드)에서 '컬러 색면'처럼 다소 비어 보인다. 브리프가 그라데이션 플레이스홀더를 허용했으니 off-brief는 아니나, 인스타의 본질적 매력(실사)이 약해지는 지점. ② 지도 캔버스가 밋밋 — 옅은 블록만 있고 거리/장소 라벨·텍스처가 없어 '진짜 지도'보다 다이어그램에 가깝다(모노 방향은 맞음). ③ 미세 조판 — notifications/settings에서 굵은 라벨과 본문 설명이 자간 없이 붙고(예 '동선 공개지도에…'), follow-seed/settings에서 스티키 CTA·fixed 탭바가 하단 콘텐츠를 덮어(리스트 하단 패딩 부족) 마지막 행이 잘린다. 풀페이지 캡처에서 fixed 요소가 콘텐츠 중앙에 떠 겹치는 건 상당 부분 캡처 아티팩트지만, settings/follow-seed의 하단 패딩 부족은 실제 여백 이슈다.

종합적으로 이 방향에서 요구되는 현행 수준을 충족하는, 응집력 있고 세련된 hi-fi다.

### 완성도 폴리시 권고(리디자인 트리거 아님 — 방향 유지, 교체 금지)

1. 사진 천장 올리기(가장 임팩트 큼): post-detail 캐러셀·map-home/profile 그리드 등 사진이 주면적을 차지하는 곳의 플레이스홀더에 최소한의 '장면 신호'를 더한다 — 단색 방향 그라데이션 위에 은은한 방사형 하이라이트/비네트, 또는 카페/공원/야경을 암시하는 형태 실루엣(컵·잎·창틀) 실루엣 오버레이. 여전히 CSS 그라데이션(외부 의존 금지) 범위 안에서. 회색 금지 규율은 유지.
2. 지도 캔버스 리얼리티: feed 히어로·map-home·onboarding의 지도에 옅은 도로 라벨(회색 텍스트)·블록 채움 톤 변주·공원 그린 틴트 영역을 더해 '다이어그램'에서 '지도'로. 핀/동선은 현행 모노 유지.
3. 조판 자간: 굵은 인라인 라벨과 뒤따르는 본문 설명 사이에 공백/줄바꿈 확보 — settings의 '동선 공개'+'지도에 방문 동선 폴리라인 표시', '팔로워'+'맞팔·팔로워에게만', notifications의 강조 이름+본문이 붙지 않게 (라벨을 별도 라인 또는 명시적 간격).
4. 하단 콘텐츠 패딩: 스크롤 화면(settings·follow-seed·profile·notifications 등)의 스크롤 컨테이너 하단에 탭바/스티키 CTA 높이만큼 padding-bottom을 줘 마지막 행이 덮이지 않게. follow-seed 마지막 추천('이산책')과 settings '권한 재설정' 행이 현재 잘림.
5. search 하단 허전함: 결과 리스트 아래 큰 공백에 '더 정확한 결과는 지도에서' 힌트나 최근 검색/스켈레톤을 둬 뷰포트 하단을 채운다.

(참고: 풀페이지 캡처에서 fixed 탭바가 중앙에 겹쳐 보이는 건 대체로 캡처 아티팩트 — 실제 390×844 뷰포트에선 하단 고정. 다만 4번 하단 패딩은 실제로 필요.)
