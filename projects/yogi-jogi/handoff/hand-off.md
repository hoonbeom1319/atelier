## 1. 지킬 계약 (디자인 의도 — 3차 확정)
- **톤 = 인스타그램 하이파이 · 다크모드**: **순검정(#000) 배경 + 근검정 카드(#121213)**, **흰 텍스트·흰 액션(primary=흰색, FAB·활성)**, 다크 헤어라인(#2c2c2f), **다크 지도(도로/블록/공원/물 다크) + 흰 핀·흰 동선**. **★사진·썸네일은 컬러 실사**(따뜻한 카페·초록 공원·도시 야경 — 회색 사진 금지, 다크 위에서 더 살아난다). 레드는 좋아요♥·미열람 닷만(로그아웃은 관례상 레드 허용). (예외: 소셜 로그인 카카오·구글 브랜드색.) 다크 전용이 기본 — 라이트는 semantic 토큰만 뒤집으면 됨.
- **홈 = 피드(`feed`) 하나** — 세로 스크롤. [지도｜리스트] 토글·전체화면 지도 없음. **각 포스트 카드 = '지도 히어로 + 작은 사진'**: 카드 상단에 **큰 지도(다녀간 동선 폴리라인 + 장소 핀)가 메인**, 그 아래 그 포스트의 **작은 컬러 사진 썸네일 행**, POI 칩·좋아요·메모·태그. *지도가 주인공, 사진은 조연.*
- **`map-home` = 장소(POI) 상세** — 한 장소의 지도 + 그곳의 포스트·사진 그리드 + 방문 친구. (피드 카드의 POI 칩·검색 장소 결과에서 진입.)
- **하단 탭 = [🏠 홈→feed][＋ 업로드][👤 프로필]**. 검색·알림은 상단 앱바.
- **토큰 2단**(primitive→semantic, 컴포넌트는 semantic만). 다크모드(P1)는 semantic 교체만. 접근성 WCAG AA · 터치 ≥44px · axe serious 0(현재 통과).
- **차별 코어**: ① 지도 히어로 피드(위) ② **다단계 업로드**(다중사진→다중 POI): ①갤러리→②자동분류+**미분류 in-place 배정**(같은 화면에서 미분류 사진 탭 선택→**카카오맵 검색** 결과로 배정, 없는 장소[할머니네집]만 **직접 핀 찍기**)→③태그·메모·공개범위→게시.
- **검색**: 장소 탭 = **카카오맵 검색 결과**(장소명·카테고리·도로명·거리). 프라이버시: 공개범위 기본=팔로워(PRD §11 A8).

## 2. 범위 (PRD §10)
- **그린 것**: 14화면 UI(모노 UI + 컬러 사진, 모바일 390×844·라이트). 백엔드 범위 밖이되 진입·온보딩 UI는 그림.
- **백엔드(대상 구현)**: 카카오 지도 SDK + 카카오 Local 검색, Supabase 카카오/구글/이메일 인증, EXIF 추출·역지오코딩, 물리 DB(개념은 PRD §5).
- **P1 자리표시**(`[범위 밖]`/`[다음 단계]`): 댓글·비공개 승인·정밀도 토글·다크모드·DM.

## 3. 만들 순서 (권장)
1. **토큰 이식**(`./token-mapping.md`, 모노 UI + 레드 액센트). 사진은 컬러(실이미지/컬러 플레이스홀더).
2. **셸** — 하단 [홈][＋][프로필] + 상단 앱바. 카카오 지도 SDK 어댑터.
3. **홈 피드** — 지도 히어로 카드(동선+핀 지도 + 컬러 사진 썸네일).
4. **장소 상세(map-home)** + **업로드 다단계**(미분류 in-place 배정: 카카오 검색 + 직접 핀).
5. **검색(카카오)·게시물 상세·프로필·알림·설정**.
6. 각 화면 상태·빈 상태(`./00-flow.md` §C) 빠짐없이. 죽은 컨트롤 0.
