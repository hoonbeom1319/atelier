## §사용자 플로우 (PRD §6 A~D)

**흐름 A — 휴가 신청(직원, 핵심 루프).**
```
login → home(직원 뷰) → [+ 휴가 신청] → request
  request: 종류·기간(반차=오전/오후)·사유 입력 → 영업일 일수 계산 → 잔액 미리보기(대기/예상잔여)
           검증: 본인 겹침? 잔액 초과? 기간 유효(종료<시작·과거)? → 위반 시 차단·경고
  → [제출] → status=pending, balance.pending+ → requests(내 신청)에 '대기' 표시
           + notifications에 승인자 대상 '승인 요청' 생성
```

**흐름 B — 승인/반려(팀장, 핵심 루프).**
```
login → home(팀장 뷰: 승인 대기 카운트) → [승인 큐] → approvals
  approvals: '대기' 탭 상단 고정 테이블(신청자·종류·기간·일수·사유·상태)
  → 행 [상세/충돌보기] → 같은 기간 팀 부재 미리보기(인라인 패널)
       ├ [승인] → status=approved, pending→used 확정, 캘린더 반영, 알림 발생
       └ [반려] → 사유 입력 모달(필수) → status=rejected, pending 복원, 사유 기록, 알림 발생
  → ['처리됨' 탭]에서 이력 조회
```

**흐름 C — 잔여 조회 · 팀 캘린더(모두).**
```
home → [잔여] → balance: 종류별 부여/사용/대기/잔여 + 사용률 + (개념)소멸 D-day
home → [팀 캘린더] → calendar: 월 그리드, 종류별 색 바+라벨, 팀 필터, 같은날 부재 N명, 대기 건
  → 부재 셀/바 클릭 → 신청 상세(권한 범위) [requests의 상세로 이동/패널]
```

**흐름 D — 취소·이력(직원) / 관리(HR).**
```
requests → 승인된 미래 휴가 행 → [취소] → 확인 → status=cancelled, used 복원, 이력 잔존
home(HR 뷰) 또는 내비 [멤버 관리] → members:
  직원 목록(이름·팀·역할·입사일·부여기준·현재 잔여)
  → 행 [부여 시드 편집] → 모달: total(부여일수)·balanceBasis(입사일/회계연도) 입력
       검증: 음수 차단 / 기존 used+pending 침범 차단 → 저장 시 remaining 재계산 + BalanceAdjustment 이력
  → [팀/승인자 배정] → teamId·managerId 설정
  → 전사 현황/이력 조회
```

**상태 전이 요약(§5 상태 머신).**
- `LeaveRequest.status`: pending → approved / rejected / cancelled. approved → cancelled(미래 한정).
- 전이마다 LeaveBalance(pending/used/remaining) 동기화 + Approval 이력 기록(불변식 G5).
- `Notification.read`: 안읽음 → 읽음(뱃지 갱신).

## §화면별 구성요소·상태 (빈·정상·검증/에러 — 3단계 spec 출처)

### SC1 login.html
- 구성: 로고·서비스명 / 이메일·비밀번호 입력 / [로그인] / **역할 시연 셀렉터(직원/팀장/HR)** + [데모 계정으로 입장].
- 동작: [로그인]/[데모 입장] → `home.html`로 이동(개념 — 실 인증 없음). 역할 선택값을 데모 진입에 반영(라벨로 표시).
- 상태: 정상(폼). 검증 경고(빈 이메일 등)는 개념 라벨. **죽은 컨트롤 0** — 모든 버튼 이동 또는 `[범위 밖]`.

### SC2 home.html (role-aware)
- 공통: 좌측 내비 + 상단바(역할 전환 셀렉터로 직원/팀장/HR 뷰 전환 — 인라인 상태 변화).
- **직원 뷰**: 내 잔여 위젯(부여/사용/대기/잔여, ←balance) / 내 신청 상태 요약(대기 N·승인 N, ←requests) / 팀 부재 미리보기(←calendar) / [+ 휴가 신청] CTA → request.
- **팀장 뷰**: 승인 대기 큐 카운트 카드 [승인 큐로] → approvals / 팀 캘린더 요약 → calendar / 같은날 부재 N명 / 내 잔여.
- **HR 뷰**: 전사 휴가 현황·승인 대기 총계 / '부여 미설정' 직원 N건 [멤버 관리로] → members / 전사 캘린더 요약.
- 상태: 역할 전환(동작) / 빈 상태(신청 0·대기 0·부여미설정 0이어도 다음 행동 CTA — 막다른 길 0).

### SC3 request.html
- 구성: 종류 선택(연차/반차/병가/경조사) / 기간(시작·종료, **반차 선택 시 오전·오후 토글 노출**) / 사유 / **잔액 미리보기 패널**(신청 일수·대기·예상 잔여) / [제출] [취소].
- 동작(인라인 JS): 종류·기간 변경 → 영업일 일수 계산 → 미리보기 갱신. 반차 토글 시 0.5일 표시. [제출] → 확인 → requests로 이동(상태 '대기' 추가).
- 검증/에러 상태: ① 본인 기간 겹침 경고·차단 ② 잔액 초과 경고·차단 ③ 기간 역순/과거 경고 ④ 반차 0.5일 표기. (시드 데이터 기반 목업.)

### SC4 approvals.html
- 구성: 탭(대기 / 처리됨) / 필터 / 테이블(신청자·종류·기간·일수·신청일·사유·상태 태그) / 행 액션 [상세] [승인] [반려] / 선택 시 **충돌 미리보기 패널**(같은 기간 팀 부재).
- 동작: [승인] → 행 상태 '승인'으로 변경(인라인) + 카운트 감소. [반려] → **사유 모달(필수)** → 제출 시 상태 '반려'. 탭 전환.
- 상태: 빈 큐(대기 0건 안내 — 막다른 길 0) / 처리됨 이력 / 반려 모달(사유 미입력 시 제출 비활성).

### SC5 calendar.html
- 구성: 월 그리드(셀=날짜, 셀 내 부재 색 바+라벨) / 월 이동 [◀ ▶] / 팀 필터 / 위젯(같은날 부재 N명·승인 대기 건) / **색 범례**(종류별).
- 동작: 월 이동(그리드 갱신) / 팀 필터(표시 갱신) / 색 바 클릭 → 신청 상세(패널/이동).
- 상태: 정상(부재 있음) / 빈 그리드(부재 0인 달 정상 표시) / 범례 항상.

### SC6 balance.html
- 구성: 종류별 카드(부여 total / 사용 used / 대기 pending / 잔여 remaining / 사용률 % / (개념)소멸 D-day·부여기준 배지).
- 동작: 종류 탭/필터(있으면). 카드 클릭 → 관련 내 신청(이동). (대부분 표시 — 읽기 중심.)
- 상태: 정상 / 잔여 0 소진 상태 명확 / 대기>0 "대기 N일 포함" 안내.

### SC7 requests.html (내 신청 + 상세/이력)
- 구성: 내 신청 테이블(종류·기간·일수·상태 태그·신청일) / 행 클릭 → **상세 패널/페이지**(상태 이력: 신청→승인/반려→취소, 타임스탬프·처리자) / 미래 승인 건 [취소] 액션.
- 동작: 행 선택 → 상세 표시(인라인 상태) / [취소] → 확인 → 상태 '취소'로 변경 + 이력 추가(잔존).
- 상태: 빈(신청 0건 안내 + [+ 신청] CTA) / 상세(이력 타임라인) / 취소 확인.

### SC8 notifications.html
- 구성: 알림 목록(종류 아이콘·메시지·시각·읽음 점) / 안읽음 뱃지 카운트 / [모두 읽음].
- 동작: 항목 클릭 → 읽음 처리(점 제거·카운트 감소) + 관련 신청으로 이동(링크). [모두 읽음] → 전체 읽음.
- 상태: 정상 / 빈(알림 0건 안내) / 안읽음 카운트 갱신.

### SC9 members.html (HR)
- 구성: 직원 테이블(이름·팀·역할·입사일·부여기준·현재 잔여·'부여 미설정' 플래그) / 행 액션 [부여 시드 편집] [팀/승인자 배정] / 전사 현황·이력 링크.
- 동작: [부여 시드 편집] → 모달(total·balanceBasis 입력, **검증: 음수 차단·기존 used+pending 침범 차단**) → 저장 시 행 잔여 갱신 + 변경 이력(BalanceAdjustment 개념). [배정] → 모달(팀·승인자).
- 상태: 정상 / 빈(직원 0명 첫 실행 안내 — 막다른 길 0) / 검증 에러(음수·하향) / RBAC(직원·팀장 진입 시 비노출 — 와이어에선 `[범위 밖: HR 전용]` 표시).

### SC10 settings.html (셸)
- 구성: 본인 프로필 카드(이름·팀·역할·입사일·부여기준 배지) / 환경설정(알림 on/off 등 — **`[범위 밖]` 개념 라벨**) / [로그아웃].
- 동작: [로그아웃] → login으로 이동. 프로필은 읽기. 환경설정 토글은 `[범위 밖]`.
- 상태: 정상(읽기) — **쓰기 기능 없음(의도적 셸)**.

## §인터랙션 모델 (구조 — spec 파생 핵심)

- **역할 전환** = 단일 선택 셀렉터(상단바). 선택 시 home 뷰·내비 가시성 변경(인라인 상태). 와이어에서도 실제로 뷰가 바뀐다.
- **승인/반려** = 행 단위 액션. 반려는 **모달 필수 입력**(사유 없이는 제출 비활성) — 단일 액션이지만 2-step(클릭→모달).
- **신청 폼** = 입력에 반응하는 실시간 계산(일수·잔액 미리보기). 반차 토글이 폼 구조를 바꿈(오전/오후 노출).
- **시드 편집(HR)** = 모달 입력 + 검증 게이트(음수·침범 차단). 단일 행 대상.
- **캘린더** = 월 단위 네비게이션(◀▶) + 필터. 셀 클릭 = 상세 진입.
- **탭** = 승인 큐(대기/처리됨), 캘린더 월 이동 등 — 같은 화면 내 상태 전환(별도 페이지 아님).
- **빈 상태** = 모든 목록형 화면(신청·큐·잔여·알림·부재·멤버)이 0건일 때 막다른 길 없이 다음 행동 CTA.

## §범위 표시 (와이어에서 `[범위 밖]`으로 명시할 것)

- 실 인증(로그인은 UI만, 백엔드 없음) · 실시간 동기화 · 알림 실 발송 · 다단계 승인/위임 · 누적/이월 자동계산 · 공휴일 자동제외 · 반반차 · CSV 내보내기 · 다크모드 · 모바일 · 환경설정 쓰기(SC10) · LeaveType 편집 UI · 계정 생성/비활성.
- 이들은 와이어에서 버튼이 있으면 `[범위 밖]` 라벨로, 없으면 미배치(조용한 누락 아님 — PRD §10에 근거).

## §열린 질문 (가정으로 닫음 — PRD §11 참조)

- 캘린더 표현(셀=날짜 내 부재 바 방식 채택 — B7). 멤버 행 타임라인은 P1.
- 과거일 소급 신청(미래·당일 기본, 과거는 경고 — B3).
- 영업일 계산은 주말 제외만(공휴일 P1 — B2).
- 잔액 모델은 4필드 표시(트랜잭션 원장은 구현 선택 — B4).
- 전부 PRD §11 가정 원장에 명시됨. 무인이라 사람 확인 없이 위 가정으로 진행.
