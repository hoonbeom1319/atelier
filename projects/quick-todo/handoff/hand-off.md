# quick-todo 디자인 인계 — 대상 프로젝트에 구현

## 목표
아래 디자인 산출물을 **대상 프로젝트의 컨벤션**으로 구현한다. 디자인은 *참고*, 흐름·기능은 *계약(필수)*.
quick-todo는 단일 사용자·로컬 전용의 가벼운 할 일 관리 앱이다: 추가 → 완료 토글 → 삭제 → 상태별 필터, 라이트/다크, 반응형(모바일 390px·데스크톱 1280px).

## 먼저 읽을 것
- 대상 프로젝트의 `CLAUDE.md` — 너의 코드 컨벤션(폴더구조·아키텍처·프레임워크·상태관리·저장 방식). **여기 맞춰 짠다.**
- 제품 요구사항 (왜·무엇의 근거):
  - `projects/quick-todo/PRD.md` — 배경·목표·기능 요구사항(동작·엣지케이스)·**데이터 모델**·범위 외. 흐름과 디자인이 *왜* 이렇게 됐는지 여기서 확인.
- 디자인 패키지 (atelier 산출물, 저장소 기준 상대경로):
  - chosen final 화면 `projects/quick-todo/screens/` — 최종 비주얼·상태·전환 (index.html 진입, main.html 메인)
  - `projects/quick-todo/handoff/index.html` — 비주얼 문서: live 임베드 + 흐름맵 + 화면별 스펙 + 검증 통과표
  - `projects/quick-todo/handoff/token-mapping.md` — semantic 토큰 → 너의 토큰 매핑
  - `projects/quick-todo/handoff/component-inventory.md` — 컴포넌트 variant/state + 사용 화면
  - `projects/quick-todo/foundation/tokens.css` — 표준 2단(primitive → semantic), 라이트/다크 양 모드 원본
  - `projects/quick-todo/00-flow.md` — 흐름 + 화면별 구성요소·상태·인터랙션 모델 = **지킬 계약**

## 지킬 것 (계약 — 임의 변경 금지)
- **화면·라우팅:** 단일 기능 화면(main). 라우팅 없이 상태로 관리. index는 진입 런처.
- **기능 동작:**
  - 추가 — `trim` 후 빈/공백 입력은 무시. 목록 끝에 추가. 입력 비우고 포커스 유지. 카운터 +1.
  - 완료 토글 — completed 반전, 완료 항목은 취소선 + muted(불투명도로 흐리지 말 것 — WCAG AA 미달). 카운터 갱신. 현재 필터 기준 재렌더.
  - 삭제 — 즉시 제거(확인창 없음). 0개면 빈 상태.
  - 필터 — 전체/미완료/완료 단일선택(뷰 상태, 데이터 필드 아님). 선택 탭 강조(aria-selected/aria-current). 매칭 0개면 필터별 빈 상태.
  - 카운터 — "남은 할 일 N개"(미완료 개수), 모든 변경에 즉시 갱신.
  - 영속 — 목록·선택 필터·테마를 로컬 저장, 재진입 복원. 저장 손상 시 빈 목록으로 안전 폴백.
- **접근성:** 키보드 전체 조작(입력 Enter 추가, 체크박스·삭제·필터·테마 포커스 가능). 라이트·다크 양 모드 WCAG AA. role·aria 라벨 유지.
- **반응형:** 390px·1280px 무파손(가로 스크롤 0). 긴 텍스트 줄바꿈.
- **토큰:** 매핑표대로 **대상 시스템 토큰에 매핑**(통째 복붙 금지). 다크 모드는 같은 역할명 값 교체.

## 범위 밖 (이번에 만들지 않음 — PRD §10)
계정·동기화·협업, 외부 연동, 마감일·우선순위·정렬·검색·태그·서브태스크, 인라인 편집·전체완료·완료모두지우기·삭제 실행취소(P1 후보). 임의로 추가하지 말 것.

## 만들 순서 (제안)
1. 토큰 매핑 반영(라이트/다크) → 2. 공통 컴포넌트(todo-item · filter-bar · input-add · theme-toggle · empty-state) → 3. main 화면 조립 → 4. 흐름·상태·영속 연결 → 5. index 진입 런처.

## 검수
- 흐름·기능이 `00-flow.md` 계약대로 동작하는지 확인(추가/토글/삭제/필터/카운터/빈상태/영속/테마).
- 라이트·다크 양 모드 대비, 390/1280 무파손.
- 참고용으로 atelier의 Playwright spec(`projects/quick-todo/wire.spec.js`)이 검증한 동작 목록을 그대로 만족하는지 대조.

## 무인 파이프라인 메모 (검수 우선순위)
이 패키지는 /forge 무인 모드로 생성됐다 — 시작 1회 인테이크 외 사람 검수가 아직 없다. 자동 게이트(형식·기능·a11y·트렌드·인계 완결성)는 통과했으나 *옳은 제품인가·최종 미감*은 미검수다. 에이전트가 사용자 대신 내린 핵심 가정은 `PRD.md` §11 및 `plan-decisions.md` 참조. 특히: 즉시 삭제(실행취소 없음), 인라인 편집 제외(P1), 시드 샘플 3건은 데모용(첫 진입 시에만).
