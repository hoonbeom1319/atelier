## §화면 목록
1. **index.html — 진입/랜딩.** 앱 진입점(폴더 entry). 앱 제목·한 줄 소개 + "할 일 목록 열기 →" 1급 링크로 main으로 보낸다. (atelier 폴더 entry 규약: index에서 클릭으로 흐름 진입.)
2. **main.html — 메인 할 일 화면(유일 기능 화면).** PRD SC1 그대로 — 입력·필터바·목록·빈 상태가 한 화면. 라우팅 없이 **상태로** 관리(추가/완료/삭제/필터/테마는 전부 이 화면 안 인라인 JS).

> 단일 기능 화면 앱이다(PRD §6). index는 entry 런처일 뿐 앱 라우트가 아니다 — 모든 todo 동작은 main 안에서 일어난다.

## §사용자 플로우 (A→B·분기)
```
[index] "할 일 목록 열기 →" 클릭 ──nav──> [main]

[main] 진입 시 입력창 자동 포커스
  · 텍스트 + Enter/"추가" ──(trim 후 빈 값이면 무시)──> 목록 끝에 추가 · 입력창 비움 · "남은 N개" +1
  · 항목 체크박스 클릭 ──> completed 반전 · 완료 스타일(취소선/흐림) · 카운터 갱신 · 현재 필터 기준 재렌더
        └ 현재 필터가 "미완료"면 완료로 바뀐 항목은 목록에서 사라짐(필터 재평가)
  · 항목 삭제(✕) 클릭 ──> 즉시 제거(확인창 없음) · 카운터 갱신 ──(목록 0개면)──> 빈 상태
  · 필터 탭(전체/미완료/완료) 클릭 ──> 해당 조건만 표시 · 탭 강조(aria-current) ──(매칭 0개면)──> 필터별 빈 상태
  · 테마 토글 클릭 ──> 라이트 ⇄ 다크 (aria-pressed 반전) · 선택 영속
[새로고침/재진입] ──> 저장된 목록 + 마지막 필터 + 마지막 테마 복원
```

## §화면별 구성요소·상태  ← (2층 기능 검증 spec이 여기서 파생)

### index.html (진입)
- **구성:** 앱 제목, 한 줄 소개, 1급 링크 `할 일 목록 열기 →`(→ main.html), (작게) 무엇을 하는 앱인지 요약.
- **인터랙티브:** 링크 1개(main으로 nav) + 테마 토글(선택 — 두면 wired). 죽은 컨트롤 0.
- **상태:** 단일 정적 상태(랜딩). 테마 토글을 두면 light/dark 전환.

### main.html (메인 — 핵심)
구성 영역(PRD SC1):
1. **헤더** — 제목 + **테마 토글**(role=button, `aria-pressed`로 light/dark).
2. **입력 영역** — 텍스트 `input`(자동 포커스, placeholder "할 일을 입력하세요") + **"추가" 버튼**. Enter도 추가.
3. **필터 바** — 탭 3개 **전체 / 미완료 / 완료**(role 구분, 선택 탭 `aria-current="true"`) + **"남은 할 일 N개" 카운터**.
4. **할 일 목록** — 각 행: `[체크박스] [할 일 텍스트] [삭제 ✕]`. 완료 행은 취소선/흐림. 촘촘한 리스트.
5. **빈 상태** — 4를 대체. (a) 전체 0개 → "첫 할 일을 추가해 보세요" (b) 필터 매칭 0개 → 필터별 메시지(예 "완료된 할 일이 없습니다"). 입력 영역(2)은 빈 상태에서도 유지.

**컴포넌트별 상태:**
- 입력창: 비어있음(placeholder) / 입력중 / 추가 직후(비워지고 포커스 유지).
- 항목 행: 미완료 / 완료(취소선+흐림). hover 시 삭제 버튼 강조(hi-fi).
- 필터 탭: 선택 / 비선택(`aria-current`).
- 테마 토글: light(`aria-pressed=false`) / dark(`aria-pressed=true`).
- 목록 영역: 항목 있음 / 빈(전체0) / 빈(필터매칭0).

## §인터랙션 모델 (구조 — 모양 아님)
- **단일 라인 입력 + Enter/버튼 추가** (멀티라인·붙여넣기 분할 없음).
- **항목당 인라인 액션**(체크 토글 + 삭제). 일괄 선택·다중선택 **없음**(toggle-all/clear-completed는 P1, §10 범위 밖 — 와이어에 두지 않음).
- **필터 = 단일 선택 세그먼트**(상호배타 3택1, view state enum). 항목 데이터 아님.
- **테마 = 2상태 토글**(light/dark).
- **삭제 = 즉시**(확인창/실행취소 없음 — F3, 즉시 제거).
- **영속:** 목록·필터·테마를 로컬 저장(인라인 JS, localStorage), 재진입 복원.

## §범위 표시
- **범위 안(이번 와이어/하이파이에 만든다):** F1 추가 · F2 완료토글 · F3 삭제 · F4 필터 · F5 카운터 · F6 빈상태 · 테마 토글 · 영속.
- **범위 밖(만들지 않음 — 두더라도 `[범위 밖]` 라벨):** 인라인 편집 · 전체완료 · 완료모두지우기 · 실행취소 · 마감일/우선순위/정렬/검색/태그 등(PRD §10). → 와이어에 이런 컨트롤을 두지 않는다(죽은 컨트롤·scope 부풀림 방지).

## §열린 질문 (무인 — 가정으로 닫음)
- **긴 텍스트 처리:** 줄바꿈 vs 말줄임 → **줄바꿈(word-break) + 행 자동 높이**로 확정(정보밀도 톤에 자연스럽고 내용 잘림 없음). 가로 스크롤 0 유지.
- **모바일 삭제 노출:** hover가 없는 터치 환경 고려 → 삭제 ✕는 **항상 노출**(hover-only 숨김 금지). hi-fi에서 hover는 강조만.
- **추가 위치:** 새 항목은 **목록 끝**에 추가(PRD F1). 정렬 없음.
- 그 외 미답은 PRD §11 가정 원장에 있음(단일사용자·로컬·MVP 최소).
