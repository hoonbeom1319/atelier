1. 화면 (chosen final — screens-refined)

2. 흐름맵 (00-flow §2 — 지킬 계약)

입력:    index → [+ FAB] → add → [저장] → index (요약 갱신)
돌아보기: index/탭 → list → (필터 칩) → [항목] → detail → [수정]→add / [삭제]→확인→list
통찰:    탭 → stats → [◀▶ 월 이동] / [수입·지출 탭] / [카테고리·멤버 드릴]
고정비:   탭 → recurring → [+ 추가]→인라인 폼→[저장] / [활성 토글] / [삭제]
하단탭:   홈 · 내역 · 통계 · 반복  (index·list·stats·recurring 공통; add·detail은 푸시 화면)

3. 화면별 스펙 (구성요소·상태·기능 책임)

화면핵심 구성요소상태기능 책임(인터랙션 모델)
index.html 홈멤버 아바타 · 이번달 카드(지출·수입·잔액) · 최근 거래(한 카드 내 행) · FAB정상 / 빈(거래 0)FAB→add, 행→detail, 하단탭. 데모 토글로 빈 상태.
add.html 입력수입/지출 세그먼트 · 금액+숫자패드 · 카테고리 아이콘 그리드 · 날짜 · 멤버 칩 · 공동/개인 · 빠른입력 · 반복 · 메모 · 저장(하단 고정)작성중 / 유효성 미충족(저장 disabled) / 유효단일선택(세그먼트·카테고리·멤버·공동개인) · 키패드 누적 · 반복 토글로 옵션 노출 · 빠른칩 프리필 · 저장→index. 유효=금액>0 & 카테고리.
list.html 내역필터 칩(기간 단일 / 카테고리·멤버·공동개인 다중) · 날짜 그룹 · 거래 행 · 빈 상태정상 / 필터 결과 0칩 토글로 목록 필터(aria-pressed) · 행→detail · 초기화.
detail.html 상세큰 금액 · 유형·카테고리·날짜·멤버·공동개인·메모·출처 · 수정·삭제기본 / 삭제 확인 시트수정→add · 삭제→확인 시트(dialog)→확인→list.
stats.html 통계월 페이저 · 요약(전월 대비) · 수입/지출 탭 · 카테고리 도넛(상위5+기타, hue 분리)+범례(1:1) · 멤버별 가로막대정상 / 데이터 없는 달월 ◀▶로 값 갱신 · 탭 전환 · 범례 선택. 색만 의존 금지(아이콘·라벨·% 병행).
recurring.html 반복추가 버튼 · 인라인 폼 · 규칙 카드(다음 발생일·활성 토글·삭제) · 빈 상태정상 / 빈추가→폼 토글 · 저장→카드 추가 · 활성/일시중지 토글 · 삭제.
