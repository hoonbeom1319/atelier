spotlog — 인계 비주얼 문서 (chosen final: screens/)

"공간(장소) 축의 인스타" — 사진을 여러 장 올리면 EXIF로 장소별 자동 그룹핑되어 '장소 카드 피드'가 된다. 다크 단일, 모바일 폭 컬럼 셸(480px, 390~1280px 무파손).

이 문서는 참고다. 흐름·기능·상태는 계약(필수). 받는 쪽은 자기 컨벤션으로 구현하되 흐름·동작을 지킨다. 함께 볼 것: handoff/hand-off.md(인계 브리프) · handoff/token-mapping.md · handoff/component-inventory.md · projects/spotlog/PRD.md · projects/spotlog/00-flow.md

흐름맵 (00-flow §플로우)

흐름 A — 업로드(핵심 루프) index ─[＋업로드]→ upload-select ─[다음(N장)]→ upload-group upload-group: · 카드 장소명 탭 → [POI 후보 팝오버] → 후보 선택 → 장소명 확정(닫힘) └ [지도에서 지정] → map-pin → 확정 → upload-group 복귀 · 미분류 사진 [이동] → 기존 카드 귀속(트레이 −1, 0이면 숨김) upload-group ─[다음]→ upload-publish ─[게시]→ index (새 카드 반영) 흐름 B — 내 기록 열람 index(피드) ─[카드 탭]→ card-detail ─[뒤로]→ index index(기록 0개) → 빈 상태 "사진을 올려 첫 장소를 기록하세요" → 업로드 유도 흐름 C — 탐색·팔로우 index ─[탐색]→ explore ─[카드/사용자 탭]→ profile ─[팔로우]→ (aria-pressed 반전, 팔로워 +1) (비공개 카드는 탐색에 노출 안 됨)

화면 (8) — chosen final 임베드 + 스펙

메인 피드 (홈)진입점

screens/index.html
  • 장소 카드 세로 피드(카드=장소+사진묶음+시간+공개표시+옵션 메모/태그). 좌표만 카드 폴백 포함.
  • 하단 글로벌 내비(홈/탐색/＋업로드/프로필). 빈 상태: 업로드 유도(막다른 길 금지).
  • 상태: #emptyDemo 토글(피드↔빈상태), 카드→card-detail.

업로드 — 사진 선택

screens/upload-select.html
  • 인스타식 다중 선택 그리드(role=checkbox, aria-checked).
  • #selCount 'N장 선택' 갱신, 0장이면 '다음' disabled → upload-group.

자동 그룹핑 결과핵심

screens/upload-group.html
  • 자동 생성 장소 카드(장소명='추정', 1탭 수정) + 미분류 트레이(이동→카드 귀속, 카운트, 0이면 숨김).
  • 장소명 탭 → POI 후보 팝오버(role=dialog '장소명 후보'): 후보 선택 확정 / 후보 0건 → '주변 등록된 장소가 없습니다' + 지도 폴백.
  • 다음 → upload-publish.

지도 + 핀 직접 지정 (목업)

screens/map-pin.html
  • 지도 목업(외부 SDK 미사용) + 핀. 핀 이동→#coord 좌표 변화, 라벨 직접 입력.
  • '이 위치로 확정' → upload-group. 실연동(카카오 등)은 handoff 계약.

메모·태그 + 공개 범위/고지

screens/upload-publish.html
  • 메모(옵션)·태그(옵션, 칩 추가). 공개 스위치 #visToggle(role=switch) 기본 비공개.
  • 토글 시 #notice 고지 변화(비공개↔'공개됩니다'). 게시 → index.

카드 상세

screens/card-detail.html
  • 대표 사진(#hero) + 썸네일 스트립(탭→대표 교체, aria-pressed) + 메모·태그·위치 목업·시간.
  • '이 카드 편집' → upload-group(보정 재진입).

탐색

screens/explore.html
  • 타인 공개 장소 카드 피드. 사용자명→profile, 카드→card-detail.
  • 빈 상태 안내(#exploreEmpty). 내비 탐색=활성.

프로필

screens/profile.html
  • 사용자 정보 + 통계(장소/팔로워/팔로잉) + 팔로우 버튼(#followBtn, aria-pressed, 팔로워 수 ±1).
  • 장소 카드 그리드(타일→card-detail). 내비 프로필=활성.
