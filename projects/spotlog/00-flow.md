# spotlog — 00-flow (인테이크 합의본)

> 입력: `PRD.md`(/forge 무인 기획 산출). 모드: /forge(무인) — 사람 OK 대신 자동 게이트 + 가정 원장.
> 대상 뷰포트: **모바일 폭 컬럼 셸, 390px ~ 1280px 무파손**(반응형 분기 아님 — 중앙 정렬 모바일 폭 컬럼을 넓은 화면에서도 유지). 다크 모드 기반. 한국어.
> 이 문서의 §화면별 구성요소·상태 + §인터랙션 모델이 **3단계 기능 검증 spec(wire.spec.js)의 출처**다.

## §화면 목록 (8 화면 — PRD SC1~SC9 매핑; SC4 후보 팝오버는 SC3 안의 오버레이 상태로 모델링)

| 파일 | 화면 | PRD | 역할 |
|---|---|---|---|
| `index.html` | 메인 피드(홈) | SC1 | 진입점. 내 장소 카드 세로 피드 + 글로벌 내비 |
| `upload-select.html` | 업로드 — 사진 선택 | SC2 | 인스타식 다중 선택 그리드 |
| `upload-group.html` | 업로드 — 자동 그룹핑 결과(핵심) | SC3+SC4 | 자동 생성 장소 카드 + 미분류 트레이 + 장소명 후보 팝오버(오버레이 상태) |
| `map-pin.html` | 지도 + 핀 직접 지정(목업) | SC5 | 핀 이동으로 좌표 확정 / 라벨 직접 입력 |
| `upload-publish.html` | 메모·태그 + 공개 범위/고지 | SC6 | 메모(옵션)·태그(옵션)·비공개 토글(기본)·공개 고지·게시 |
| `card-detail.html` | 카드 상세 | SC7 | 한 장소 카드 전체(사진 묶음·메모·태그·위치 목업·시간) |
| `explore.html` | 탐색 | SC8 | 타인 공개 카드 피드 |
| `profile.html` | 프로필 | SC9 | 사용자 카드 모음 + 팔로우 버튼 |

진입점: `index.html`. 모든 화면은 글로벌 내비(홈·탐색·＋업로드·프로필)로 상호 연결.

## §사용자 플로우 (A→B·분기 — PRD §6)

**흐름 A — 업로드(핵심 루프, S1~S3):**
```
index ─[＋업로드]→ upload-select ─[다음(N장 선택)]→ upload-group
  upload-group:
    · 카드 장소명 탭 → [후보 팝오버(오버레이)] → 후보 선택 → 장소명 확정(닫힘)
                                              └ [지도에서 지정] → map-pin → 확정 → upload-group 복귀
    · 미분류 사진 [이동] → 기존 카드로 귀속(트레이 카운트 −1)
  upload-group ─[다음]→ upload-publish ─[게시]→ index (새 카드 반영)
```

**흐름 B — 내 기록 열람(S4):**
```
index(피드) ─[카드 탭]→ card-detail ─[뒤로]→ index
index(기록 0개) → 빈 상태 "사진을 올려 첫 장소를 기록하세요" → [＋업로드]
```

**흐름 C — 탐색·팔로우(S5):**
```
index ─[탐색]→ explore ─[카드/사용자 탭]→ profile ─[팔로우]→ (aria-pressed 반전, 팔로워 수 +1)
explore(결과 0건) → 빈 상태 안내
```

## §화면별 구성요소·상태 (3단계 spec의 출처 — 빈·정상·오버레이·토글)

### index.html (메인 피드)
- **구성:** 헤더(앱명), 장소 카드 리스트(`role=listitem`), 글로벌 하단 내비(홈·탐색·＋업로드·프로필 = `<a href>`), 빈 상태 데모 토글(`#emptyDemo`, [데모] 라벨).
- **카드 1개:** 장소명(`.place-name`), 주소/거리, 사진 묶음(대표+추가 N), 방문 시간 범위, 메모·태그(있으면), 공개 표시(공개/비공개).
- **상태:** 정상(시드 카드 3개) / 빈 상태(`#feedEmpty`, 데모 토글로 표시 — "사진을 올려 첫 장소를 기록하세요" + ＋업로드 CTA). 좌표만 카드 1개(장소명 자리에 사용자 라벨/동네 폴백) 포함.
- **인터랙션:** 카드 탭 → `card-detail.html`. 내비 링크 4개. 빈 상태 데모 토글(상태 변화).

### upload-select.html (사진 선택)
- **구성:** 헤더(뒤로 → index), 사진 그리드(셀 = `role=checkbox`, aria-checked), 선택 카운터(`#selCount` "0장 선택"), 다음 버튼(`#nextBtn`, 선택 0이면 disabled).
- **상태:** 사진 선택/해제 → 셀 aria-checked 토글 + 카운터 증감 + 다음 버튼 활성/비활성.
- **인터랙션:** 다중 선택(체크박스 토글), 다음 → `upload-group.html`.

### upload-group.html (자동 그룹핑 결과 — 핵심 + SC4 팝오버 오버레이)
- **구성:** 헤더(뒤로), 자동 생성 장소 카드 리스트(각 카드: 장소명 버튼 `.edit-place`(=‘추정’ 표기 + 1탭 수정), 사진 묶음, 시간범위), **미분류 트레이 섹션**(`#unclassified`, 사진들 + 각 [이동] 버튼 `.move-photo`), 다음 버튼(→ upload-publish), 장소명 후보 팝오버(`#poiPopover`, 기본 hidden).
- **상태:**
  - **장소명 후보 팝오버(SC4):** 카드 장소명 버튼 탭 → `#poiPopover` 표시(aria-expanded=true), 후보 목록(상호+주소+거리). 후보 선택 → 그 카드 장소명 텍스트 갱신 + 팝오버 닫힘. "지도에서 직접 지정" → `map-pin.html`. 후보 0건 케이스: 데모 카드 1개는 후보 0 → "지도에서 지정" 폴백만.
  - **미분류 트레이:** [이동] 버튼 → 해당 사진이 첫 카드로 귀속(트레이에서 사라짐) + 트레이 카운트 −1. 트레이 0건이면 섹션 숨김(`#unclassified[hidden]`).
- **인터랙션 모델:** 인라인 편집(장소명) + 팝오버 단일 선택 + 항목 이동(트레이→카드). 다음 → `upload-publish.html`.

### map-pin.html (지도 핀 — 목업)
- **구성:** 헤더(뒤로 → upload-group), 지도 목업 영역(`#mapArea`), 핀(`#pin`), [핀을 위로/아래로 이동] 컨트롤(좌표 텍스트 `#coord` 갱신), 라벨 직접 입력(`#labelInput`), 확정 버튼(→ upload-group).
- **상태:** 핀 이동 버튼 → `#coord` 좌표 텍스트 변화. 라벨 입력 → 값 반영.
- **인터랙션:** 핀 이동(상태 변화), 확정 → `upload-group.html`.

### upload-publish.html (메모·태그·공개·게시)
- **구성:** 헤더(뒤로), 요약(카드 N개), 메모 textarea(`#memo`, 옵션), 태그 입력(`#tagInput` + 추가 버튼 → 태그칩 추가), **공개 범위 토글**(`#visToggle`, `role=switch` aria-checked, **기본 비공개**), **"이 장소가 공개됩니다" 고지**(`#notice`, 비공개일 땐 "비공개로 게시됩니다"로 변화), 게시 버튼(→ index).
- **상태:** 메모 입력 / 태그 추가(칩 생성, 카운트) / 공개 토글(aria-checked 반전 + 고지 문구 변화). 
- **인터랙션:** 폼 입력 + 스위치 토글 + 칩 추가. 게시 → `index.html`.

### card-detail.html (카드 상세)
- **구성:** 헤더(뒤로 → index), 장소명/주소, 사진 묶음 전체(그리드), 방문 시간, 메모·태그, 위치 목업(`#miniMap`), 본인 카드 편집 진입(`.edit-card` → [범위 밖] 라벨 또는 upload-group). 공개 범위 표시.
- **상태:** 사진 썸네일 탭 → 대표 사진 교체(`#hero` 변화).
- **인터랙션:** 뒤로, 사진 썸네일 전환(상태 변화), 편집(범위 밖 또는 와이어드).

### explore.html (탐색)
- **구성:** 헤더, 타인 공개 카드 피드(카드 탭 → card-detail, 사용자명 탭 → profile), 글로벌 내비, 결과 0건 빈 상태 데모 토글(`#exploreEmptyDemo`).
- **상태:** 정상(시드 타인 카드) / 빈 상태(데모 토글).
- **인터랙션:** 카드 → card-detail, 사용자 → profile, 빈 상태 토글.

### profile.html (프로필)
- **구성:** 헤더, 사용자 정보(핸들·표시명·아바타), 팔로워/팔로잉 수(`#followerCount`), **팔로우 버튼**(`#followBtn`, aria-pressed), 카드 그리드(탭 → card-detail), 글로벌 내비.
- **상태:** 팔로우 토글 → aria-pressed 반전 + 버튼 텍스트(팔로우↔팔로잉) + 팔로워 수 ±1.
- **인터랙션:** 팔로우 토글(상태 변화), 카드 → card-detail, 내비.

## §인터랙션 모델 (구조 — 모양 아님)

- **다중 선택:** 사진 선택(upload-select) — 체크박스형 다중, 카운터 + 일괄 '다음'.
- **인라인 편집 + 단일 선택 팝오버:** 장소명(upload-group) — 탭하면 후보 팝오버, 그 중 하나 선택해 확정. 후보 0건이면 지도 폴백.
- **항목 이동:** 미분류 사진 → 카드 귀속(이동 액션 = 버튼; PRD B4 '드래그 vs 액션'은 와이어에선 버튼 액션으로 모델, hi-fi에서 마이크로인터랙션). ≤3탭(E1).
- **토글/스위치:** 공개 범위(switch, 기본 비공개) · 팔로우(pressed) · 빈 상태 데모.
- **사진 전환:** 카드 상세에서 썸네일 탭 → 대표 교체(단일 활성).

## §범위 표시

- **범위 안(이번):** 위 8 화면 + 흐름 A/B/C, 자동 그룹핑 결과·미분류 보정·후보 팝오버·핀 목업·공개 토글·팔로우.
- **범위 밖(`[범위 밖]` 라벨 또는 미구현):** 댓글·좋아요·DM·푸시 / 지도 뷰 탐색 / Trip 묶음 / 검색(전역) / 설정 / 알림 / 계정 인증 흐름 / 라이트 모드 토글. 와이어에 노출되면 `[범위 밖]`으로 명시(죽은 컨트롤 0).
- **목업:** 지도·역지오코딩 POI·EXIF는 시드/목업(외부 의존성 없는 독립 HTML). 실연동은 handoff 계약.

## §열린 질문 (무인 — 가정으로 닫음, 사람 검수 우선순위)

- **Q1.** 미분류 사진 이동을 '드래그' vs '버튼 액션' 중 무엇으로? → 가정: 와이어/hi-fi는 **버튼 액션**(모바일 드래그 사용성 리스크). ≤3탭만 만족(E1). [PRD B4]
- **Q2.** 위치 정밀도 흐리기 UX 범위(비공개 토글만 vs 동네 단위 슬라이더)? → 가정: MVP는 **비공개 토글+고지 필수**, 정밀도 슬라이더는 P1. [PRD A5]
- **Q3.** 라이트 모드 동시 제공? → 가정: **다크 단일**(인테이크 "다크 모드 기반"). 라이트는 P1. [PRD A6]
- **Q4.** 좌표만 카드의 장소명 자리 폴백 표기(라벨 vs 동네 vs 좌표)? → 가정: **사용자 라벨 우선, 없으면 동네/좌표** 폴백. [PRD §7 SC1]
