## §사용자 플로우 (A→B · 분기)

```
[최초]  S1(PIN 설정) ──입장──▶ S2(빈 목록)
[입장]  S1(PIN 입력) ──올바름──▶ S2          └─틀림─▶ S1(에러 표시, 재시도)
[일상]  S2 ──추가(빠른입력, 텍스트만)──▶ S2(항목 +1, 맨 위)
        S2 ──항목 체크──▶ S2(완료됨으로 이동: 진행중 −1 / 완료 +1)
        S2 ──완료항목 체크해제──▶ S2(진행중으로 복귀)
        S2 ──항목 삭제──▶ S2(목록에서 제거, 수 −1)
        S2 ──항목 탭──▶ S3 ──저장──▶ S2(내용·날짜 반영)
        S3 ──취소──▶ S2(변경 없음)
        S3 ──삭제──▶ S2(제거)
```

**분기 표:**
- **S1 입장:** 올바른 PIN → S2 / 틀린 PIN → 에러(같은 화면 재시도).
- **추가는 1경로:** S2 하단 빠른 입력(텍스트만, 날짜 없이 즉시). **날짜는 항목을 눌러 S3 편집에서 붙인다**(전용 ＋날짜 버튼 없음 — 3층 판단으로 제거, 중복이라). 메모형이 기본, 날짜는 나중에 얹는 흐름.
- **항목 상태:** 진행 중 ⇄ 완료됨(체크 토글). 완료는 사라지지 않고 "완료됨" 영역에 쌓임(흔적).

## §화면별 구성요소 · 상태

### S1. PIN 입장
- **구성요소:** 앱 타이틀("우리집" 류) / PIN 입력(숫자) / 입장 버튼 / 에러 메시지 영역 / [최초 모드] 안내 문구.
- **상태:**
  - *기본(입력 대기)* — 빈 PIN, 입장 버튼.
  - *에러* — 틀린 PIN 입력 시 메시지 노출 + 재시도.
  - *최초 설정 모드* — "우리집 PIN을 정해주세요" (설정 후 S2 빈 목록).
- **인터랙션 모델:** 단일 입력 → 검증 → 이동. (와이어에선 데모 PIN으로 올바름/틀림 분기 시뮬레이트.)

### S2. 할 일 목록 (메인) ★
- **구성요소:** 헤더(공간 이름) / **진행 중 목록**(항목: 체크박스 + 내용 + 날짜 배지(있으면) + 삭제, *단순 나열* — 부부 진행중은 보통 소수라 그룹 불필요) / **완료됨 영역**(접이식: `완료됨 N개 ▸` 기본 접힘 → 펼치면 흔적 + **완료 비우기**(정리)) / **하단 빠른 추가 입력**(텍스트 + 추가 버튼 — 날짜 단축 없음) / (선택) 동기화·오프라인 표시.
- **상태:**
  - *빈* — 할 일 0개. **따뜻한 빈 상태 카피**("우리집 첫 할 일을 남겨보세요" 류).
  - *로딩* — 동기화 중(스켈레톤/스피너).
  - *정상* — 진행 중 + 완료됨 함께.
  - *에러* — 오프라인 배너(끊겨도 안 깨짐).
- **인터랙션 모델:** **단일 목록, 다중선택 없음.** 항목별 독립 동작(체크/삭제/탭편집). 빠른 추가는 인라인. 날짜·편집은 S3로.

### S3. 할 일 추가·편집
- **구성요소:** 내용 입력(필수) / **날짜 추가 토글 → 날짜 선택**(선택) / 저장 / 취소 / [편집 모드] 삭제.
- **상태:**
  - *추가 모드* — 빈 입력, 날짜 off 기본.
  - *편집 모드* — 기존 내용·날짜 채워짐, 삭제 노출.
  - *검증* — 빈 내용 저장 시도 → 막음(저장 비활성/경고).
- **인터랙션 모델:** 폼 입력 → 저장 시 S2 반영 / 취소 시 변경 없음. 날짜는 토글로 on/off(선택사항임을 구조로 표현).

---

## §인터랙션 모델 (구조 결정 — 검증 대상)

- **단일 공유 목록.** 카테고리·다중목록 없음(범위 밖).
- **항목 동작은 독립적** — 다중선택·일괄액션 없음. 각 항목: 체크 토글 / 삭제 / 탭→편집.
- **완료 = 토글 + 보존 + 접기.** 체크하면 완료됨으로 *이동*(삭제 아님), 다시 체크하면 복귀. 완료시각이 흔적의 근거. **완료됨은 기본 접힘(개수만)** + **완료 비우기**로 정리 — 흔적이 무한 증식해도 진행 중 시야를 안 가린다. (스케일 대응은 *완료 누적*만 손댄다; 진행 중은 단순 나열 유지 — 3층 판단으로 "최소" 선택.)
- **추가 1경로 + 편집에서 날짜** — 생성은 빠른 입력(텍스트만). 날짜는 항목을 눌러 S3 편집에서 얹는다. (전용 ＋날짜 버튼은 중복이라 제거 — 3층 판단.)
- **날짜는 1급 옵션** — 있어도/없어도 같은 목록에 공존(메모형·일정형).

## §범위 표시 (와이어에서 처리)

- **동기화(F5)는 백엔드 계약** — 정적 와이어(file://·백엔드 없음)에서 *두 기기 실시간 반영*은 시뮬레이트 불가. 와이어에선 **단일 기기 인터랙션**만 검증하고, 동기화는 `[범위 밖(백엔드 계약)]`으로 표시. 오프라인/로딩 상태는 화면으로 표현. (구현·검수는 대상 프로젝트에서.)
- **PIN 보안 검증** — 와이어는 데모 PIN으로 분기만 시뮬레이트. 실제 PIN 저장·검증은 `[범위 밖(개발 단계)]`.
- **범위 밖 기능(라벨로 명시):** 담당자 / 알림·푸시 / 월간 캘린더 뷰 / 카테고리 / 반복·첨부·댓글 / undo / 완료 통계 화면.
