# 🎨 Atelier

> 터미널 안의 디자인 아틀리에 — 브리프를 받아 손으로 짓는다.

**Atelier**는 `claude.ai/design`을 **로컬 터미널에서 그대로 대체**하는 작업장이다.
웹에 들어가지 않고, PRD 한 장에서 와이어프레임 → 디자인 시스템 → 하이파이까지 단계별로 짓고,
완성본을 그대로 다른 Claude Code 프로젝트(앱 레포)에 넘겨 개발에 쓴다.

여기서 만드는 건 "최종 앱 코드"가 아니라 **개발에 넘길 디자인 산출물**이다.

---

## 왜 만들었나

`claude.ai/design`의 결과물이 좋은 건 모델이 특별해서가 아니다. 웹과 터미널은 **같은 Claude 모델**을 쓴다.
진짜 차이는 그 웹에 박힌 **"생성 방식(하니스)"** — 만들고 → 렌더 검증 → 다시 고치는 루프, 단계별 진행, 클릭되는 프로토타입이다.

Atelier는 그 하니스를 로컬에 옮겨 담았다. 웹 왕복 없이 터미널에서 끝낸다.

---

## 어떻게 동작하나

`/design` 스킬 하나가 전체 파이프라인을 구동한다.

```
PRD ─▶ ① 인테이크 ─▶ ② 와이어(클릭됨) ─▶ ③ ▶흐름 테스트
                                                   │
        ⑦ handoff ◀─ ⑥ ▶인터랙션 테스트 ◀─ ⑤ 하이파이 ◀─ ④ 디자인 시스템
```

- **단계마다 사용자 OK**를 받고 넘어간다. 한 번에 다 쏟지 않는다.
- **와이어는 진짜 클릭된다** — 페이지끼리 `<a href>`로 링크해 브라우저에서 흐름을 직접 타본다.
- **테스트는 두 군데** — 흐름 검증은 와이어 직후(③), 느낌(애니메이션·전환) 검증은 하이파이 직후(⑥).
- **에이전트 동적 편성** — 화면이 많은 단계는 브리프를 읽고 빌더를 병렬로 띄우고, **만들지 않은 독립 에이전트**가 적대적으로 검증한다.

---

## 구조

Atelier는 단일 프로젝트가 아니라 **여러 프로젝트를 담는 작업장**이다.

```
atelier/                            # (레포. 로컬 폴더명은 달라도 무관)
├── CLAUDE.md                       # 작업장 공통 규칙 (형식·품질의 원본)
├── .claude/skills/design/          # /design — 파이프라인 하니스
└── projects/
    ├── my-first-app/
    │   ├── PRD.md                  # 입력
    │   ├── STATUS.md               # 이 프로젝트가 지금 어느 단계인지
    │   ├── 00-flow.md              # 화면·플로우 합의본
    │   ├── wireframe/              # 클릭되는 lo-fi (*.html)
    │   ├── foundation/             # tokens.css + colors/typography
    │   ├── components/             # *.html
    │   └── screens/                # 하이파이 (*.html)
    ├── my-second-app/
    └── my-third-app/
```

- **프로젝트는 서로 독립.** 토큰·산출물을 공유하지 않는다.
- **`STATUS.md`가 프로젝트의 현재 위치.** 파일만으로는 알 수 없는 것(예: "흐름 테스트를 통과시켰는가")을 기록한다.

---

## 쓰는 법

작업장 폴더에서 새 세션을 열고:

```
/design
```

- 처음이면 → `projects/`가 비어 있으니 **새 프로젝트 생성**부터 안내한다.
- 진행 중이면 → 프로젝트 **대시보드**를 보여주고, 고른 프로젝트의 `STATUS.md`에서 멈춘 단계부터 이어간다.

PRD는 `projects/<name>/PRD.md`로 넣거나 첫 메시지에 붙여넣으면 된다.

---

## 산출물 형식

- 화면·컴포넌트마다 **외부 의존성 없는 독립 실행 HTML 1개.** 브라우저로 바로 열린다.
- 각 파일 첫 줄에 카드 마커: `<!-- @dsCard group="Wireframe|Foundation|Components|Screens" -->`
- 이 마커는 `claude.ai/design`과 호환 — 원하면 `/design-sync`로 웹에 그대로 업로드할 수도 있다.

---

## 품질 기준 (render-check)

산출물을 만들 때마다 스스로 점검하고, 걸리면 통과할 때까지 고친다:

| 항목 | 본다 |
|---|---|
| **thin** | 내용이 빈약한가 (상태·variant·실데이터가 충분한가) |
| **bad** | 레이아웃이 깨졌나 (정렬·간격·오버플로우·반응형) |
| **variantsIdentical** | variant·화면이 사실상 똑같아 보이나 |
| **off-brief** | `00-flow.md`/PRD에서 벗어났나 |

---

## handoff

완성된 디자인을 앱 레포로 넘길 때:

1. **대상 프로젝트의 CLAUDE.md를 먼저 읽고** 그 컨벤션(예: FSD + React)으로 변환한다.
2. 토큰은 대상의 토큰 시스템에 **매핑**한다. 통째 복붙하지 않는다.
