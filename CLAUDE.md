# Atelier — 로컬 claude.ai/design 작업장

이 폴더는 `claude.ai/design`을 **로컬 터미널에서 그대로 대체**하는 작업장이다.
웹에 들어가지 않고 여기서 PRD를 받아 디자인 산출물을 단계별로 만들고, 완성본을 다른 Claude Code 프로젝트에 넘긴다.

> **이 폴더는 단일 프로젝트가 아니라 여러 프로젝트를 담는 "작업장"이다.**
> 실제 디자인 작업물은 전부 `projects/<프로젝트명>/` 안에서 일어난다.
> 여기서 만드는 건 "최종 앱 코드"가 아니라 **개발에 넘길 디자인 산출물**이다.

---

## 폴더 구조

```
atelier/                            # 작업장 루트 (= 로컬 claude.ai/design. 로컬 폴더명은 달라도 무관)
├── CLAUDE.md                       # 이 파일 — 작업장 공통 규칙
├── .claude/skills/design/          # /design 스킬 (파이프라인 구동 하니스)
└── projects/
    ├── my-first-app/
    │   ├── PRD.md                  # 입력
    │   ├── STATUS.md               # ★ 이 프로젝트가 어느 단계인지 기록 (스킬이 갱신)
    │   ├── 00-flow.md              # 1단계 산출: 화면·플로우 합의본
    │   ├── wireframe/*.html        # 클릭되는 lo-fi
    │   ├── foundation/             # tokens.css + colors/typography
    │   ├── components/*.html
    │   └── screens/*.html          # 하이파이
    ├── my-second-app/
    └── my-third-app/
```

- **프로젝트는 서로 독립.** 토큰·산출물을 공유하지 않는다.
- **`STATUS.md`가 프로젝트의 "현재 위치"다.** 파일 존재 여부만으로는 알 수 없는 것(예: "흐름 테스트를 사용자가 통과시켰는가")을 여기에 기록한다. 작업장에 들어오면 이 파일들이 곧 claude.ai/design의 프로젝트 목록 역할을 한다.

## 파이프라인 (각 프로젝트 안에서 PRD → handoff)

각 단계는 **사용자의 "OK"를 받고** 다음으로 넘어간다. 넘어갈 때마다 `STATUS.md`를 갱신한다.

1. **인테이크** — PRD를 읽고 화면 목록 / 플로우 / **화면별 구성요소·상태** / **인터랙션 모델** / 범위를 `00-flow.md`로 정리·합의. (3단계 기능 검증 spec이 여기서 파생된다.)
2. **와이어프레임 (lo-fi)** — 흑백 골격. **페이지끼리 진짜 `<a href>`로 링크해 클릭하면 이동하게.** "정적 화면"이 아니다 — **상태는 진짜로 변한다**(선택·토글·카운터, 밋밋하게. 인라인 JS 허용). **죽은 컨트롤 금지**: 모든 인터랙티브 요소는 와이어드 되거나 `[범위 밖]`이거나.
3. **▶ 흐름·기능 검증 (3층)** — **(자동/1·2층)** Playwright로 green까지 루프 — ①내비게이션(A→B / 막다른 길 0 / 범위 안 도달) ②**기능 완결성**(화면이 책임지는 동작이 실제로 작동하나 / 죽은 컨트롤 0 / 상태가 실제로 변하나). spec은 `00-flow.md`의 플로우+구성요소·상태+인터랙션 모델에서 파생. → **(사람/3층)** "맞는 흐름인가·빠진 건 없나" 판단. 수정은 **색칠 전에**.
4. **디자인 시스템** — 흐름 확정 후 `tokens.css` + 기본 컴포넌트 확정.
5. **하이파이 (hi-fi)** — 토큰 입힌 실제 화면. **링크 구조 + 상태 동작 유지** 위에 전환·애니메이션·상태(로딩/빈/에러)를 입힌다.
5.5 **리디자인 (선택)** — hi-fi가 "와이어 + 색"에 머물러 도약이 필요할 때만. 히스토리(00-flow·wireframe·screens·tokens)를 다 읽고 **흐름·기능은 지킨 채 비주얼 언어(구도·레이아웃·타이포·색·모션)를 재해석**. 원본 안 덮고 후보 폴더(`screens-<variant>/`)로. 각 후보는 **기능 spec(2층)을 그대로 통과**해야(가드레일). 사용자가 **chosen final**을 고른다.
6. **▶ 인터랙션 검증** — **(자동)** 3단계 기능 spec(2층)을 **chosen final**에 재실행해 동작 회귀 없는지 green까지 → **(사람)** 전환·애니메이션 *느낌* 판단.
7. **handoff** — **chosen final**을 *패키지 + 인계 프롬프트*로 묶는다. `handoff/`에 비주얼 문서(`index.html`)·토큰 매핑표·컴포넌트 인벤토리 + ★**인계 프롬프트 `hand-off.md`**(받는 쪽 Claude Code에 붙여넣는 브리프 — 참고 파일·지킬 계약·만들 순서 명시). **atelier는 대상 코드를 직접 짜지 않는다** — 이 프롬프트를 넘기면 대상 Claude가 자기 컨벤션으로 구현(특정 프레임워크에 안 묶임). 직접 호출 가능(재-handoff·다중 대상).

> **검증은 3층:** 내비게이션(기계) + 기능 완결성(기계) + 느낌·판단(사람). "기계는 내비게이션만"이 아니다 — 화면이 책임지는 **동작·상태 변화도 기계가 green까지** 본다. 비주얼·느낌·"옳은 흐름인가"만 사람 몫.
> **3단계는 동작이 *되나*(밋밋한 상태 변화), 6단계는 그게 *좋게 느껴지나*(전환·타이밍).** 같은 기능 spec을 `getByRole`·`aria-*`·텍스트로 짜서 3·6 양쪽에 돌린다.

### 묶음(청크) 진행 순서 — 큰 PRD

화면이 많으면 묶음으로 나누되, **두 규칙이 다른 높이에서 동시에 성립한다.**

- **① 단계(세로)로 자른다** — 한 **묶음**을 와이어→토큰→하이파이 끝까지 내리지 **않는다.** `tokens.css`는 전체 공유 토대라 모든 묶음 구조 확정 뒤 한 번에 만든다.
- **② 단계 *안에선* 묶음마다 게이트** — 와이어 단계에서 전 화면을 한꺼번에 쏟고 "다 됐어, 전체검증해줘"로 가지 **않는다.** 묶음1 와이어 → 기계 green + 사람 OK → 묶음2 → … 끊어서 내린다. 묶음1 구조가 검증된 뒤 묶음2가 물려받아야 품질이 오른다. (①·②는 충돌이 아니라 다른 높이의 규칙.)

```
와이어: 묶음1 검증·OK → 묶음2 검증·OK → 묶음3 검증·OK
   → 전체 도달성 크롤 1회(묶음 간 링크 연결) → tokens.css(1회) → 하이파이: 묶음별 → 인터랙션 검증 → handoff
```

묶음N 단독 검증 시 다음 묶음 링크는 **스텁/`[범위 밖]`**으로 막다른 길을 막고, 묶음 간 연결은 와이어 배리어에서 한 번에 잇는다.

## 결과물 형식 (claude.ai/design과 동일하게)

- 화면·컴포넌트마다 **외부 의존성(외부 라이브러리) 없는 독립 실행 HTML 1개.** 브라우저로 바로 열려야 한다. (상태 변화용 **인라인 JS는 허용** — 금지는 외부 의존성이지 JS가 아니다.)
- 각 파일 첫 줄에 카드 마커: `<!-- @dsCard group="Wireframe|Foundation|Components|Screens" -->`
- 디자인 토큰은 그 프로젝트의 `foundation/tokens.css`에 CSS 변수로 모으고 전 화면이 공유. **표준 2단 구조**: ①primitive(`--coral-500`·`--gray-50..900`, 중립은 숫자 램프 필수) → ②semantic 역할명(`--color-primary`·`--color-text`·`--color-surface`…, 컴포넌트는 이것만 사용). **브랜드 단어로 토큰명 짓지 않기**(`--c-coral` X — 코랄은 값이지 이름이 아님). 제품 전용 토큰(`--map-*` 등)은 파운데이션이 아니라 해당 컴포넌트에.
- 하이파이는 **그 프로젝트의 대상 뷰포트**(00-flow에서 정함 — 모바일 390px·데스크톱·대시보드 등) 안에서. 모바일이 기본이 아니다.
- 각 화면 폴더에 `index.html`(진입점)을 둬서 거기서부터 클릭으로 흐름을 탈 수 있게.

## 품질 루프 (모든 산출물에 적용)

만들 때마다 스스로 점검하고, 걸리면 **통과할 때까지** 고친다:

1. **thin** — 내용이 빈약한가 2. **bad** — 레이아웃이 깨졌나
3. **variantsIdentical** — variant·화면이 사실상 똑같나 4. **off-brief** — `00-flow.md`/PRD에서 벗어났나
5. **deadControl** — 인터랙티브한데 와이어드 안 됨(범위 밖 라벨도 없음) 6. **stateInert** — 눌렀는데 상태가 안 변함

## handoff

- **handoff는 "코드 변환"이 아니라 "인계 프롬프트 전달"이다.** chosen final을 `handoff/` 패키지(비주얼 문서·토큰 매핑표·컴포넌트 인벤토리 + **PRD가 있으면 `PRD.md`도 참조로 포함** — `/plan` 산출물)로 묶고, **`hand-off.md`** = 받는 쪽 Claude Code에 붙여넣는 브리프(참고 파일·지킬 계약·만들 순서)를 만든다. 대상 프로젝트의 Claude가 그걸 보고 **자기 컨벤션으로** 짠다. **atelier는 대상 코드를 직접 짜지 않는다 — 특정 프레임워크·아키텍처에 묶이지 않는다.**
- 토큰은 대상 토큰 시스템에 **매핑**한다. 통째 복붙 금지.

## 참고
