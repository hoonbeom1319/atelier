# AI Design — 로컬 claude.ai/design 작업장

이 폴더는 `claude.ai/design`을 **로컬 터미널에서 그대로 대체**하는 작업장이다.
웹에 들어가지 않고 여기서 PRD를 받아 디자인 산출물을 단계별로 만들고, 완성본을 다른 Claude Code 프로젝트에 넘긴다.

> **이 폴더는 단일 프로젝트가 아니라 여러 프로젝트를 담는 "작업장"이다.**
> 실제 디자인 작업물은 전부 `projects/<프로젝트명>/` 안에서 일어난다.
> 여기서 만드는 건 "최종 앱 코드"가 아니라 **개발에 넘길 디자인 산출물**이다.

---

## 폴더 구조

```
ai-design/                          # 작업장 루트 (= 로컬 claude.ai/design)
├── CLAUDE.md                       # 이 파일 — 작업장 공통 규칙
├── .claude/skills/design/          # /design 스킬 (파이프라인 구동 하니스)
└── projects/
    ├── my-first-app/
    │   ├── PRD.md                  # 입력
    │   ├── STATUS.md               # ★ 이 프로젝트가 어느 단계인지 기록 (스킬이 갱신)
    │   ├── 00-flow.md              # 1단계 산출: 화면·플로우 합의본
    │   ├── wireframe/*.html        # 클릭되는 lo-fi
    │   ├── foundation/             # tokens.css + colors/typography
    │   ├── components/*.html
    │   └── screens/*.html          # 하이파이
    ├── my-second-app/
    └── my-third-app/
```

- **프로젝트는 서로 독립.** 토큰·산출물을 공유하지 않는다.
- **`STATUS.md`가 프로젝트의 "현재 위치"다.** 파일 존재 여부만으로는 알 수 없는 것(예: "흐름 테스트를 사용자가 통과시켰는가")을 여기에 기록한다. 작업장에 들어오면 이 파일들이 곧 claude.ai/design의 프로젝트 목록 역할을 한다.

---

## 파이프라인 (각 프로젝트 안에서 PRD → handoff)

각 단계는 **사용자의 "OK"를 받고** 다음으로 넘어간다. 넘어갈 때마다 `STATUS.md`를 갱신한다.

1. **인테이크** — PRD를 읽고 화면 목록 / 플로우 / 핵심 상태를 `00-flow.md`로 정리·합의.
2. **와이어프레임 (lo-fi)** — 흑백 골격. **페이지끼리 진짜 `<a href>`로 링크해 클릭하면 이동하게.**
3. **▶ 흐름 테스트** — 사용자가 브라우저에서 눌러보며 확인: A→B 이동 / 빠진 화면 / 막다른 길 / 뒤로가기. 수정은 **색칠 전에** 끝낸다.
4. **디자인 시스템** — 흐름 합의 후 `tokens.css` + 기본 컴포넌트 확정.
5. **하이파이 (hi-fi)** — 토큰 입힌 실제 화면. 링크 구조 유지 + 전환·애니메이션·상태(로딩/빈/에러).
6. **▶ 인터랙션 테스트** — 사용자가 눌러보며 *느낌* 확인: 전환 자연스러움 / 애니메이션 타이밍 / 피드백 / 상태 전이.
7. **handoff** — 대상 프로젝트의 CLAUDE.md를 읽고 그 컨벤션으로 변환해 넘긴다.

> **흐름 검증은 lo-fi(3), 느낌 검증은 hi-fi(6).** 다른 테스트다. 섞지 않는다.
> 큰 PRD는 **플로우 묶음 단위로** 2~6단계를 반복한다. 수십 화면 한 번에 금지.

## 결과물 형식 (claude.ai/design과 동일하게)

- 화면·컴포넌트마다 **외부 의존성 없는 독립 실행 HTML 1개.** 브라우저로 바로 열려야 한다.
- 각 파일 첫 줄에 카드 마커: `<!-- @dsCard group="Wireframe|Foundation|Components|Screens" -->`
- 디자인 토큰은 그 프로젝트의 `foundation/tokens.css`에 CSS 변수로 모으고 전 화면이 공유.
- 하이파이는 실제 디바이스 뷰포트(예: 모바일 390px) 안에서.
- 각 화면 폴더에 `index.html`(진입점)을 둬서 거기서부터 클릭으로 흐름을 탈 수 있게.

## 품질 루프 (모든 산출물에 적용)

만들 때마다 스스로 점검하고, 걸리면 **통과할 때까지** 고친다:

1. **thin** — 내용이 빈약한가 2. **bad** — 레이아웃이 깨졌나
3. **variantsIdentical** — variant·화면이 사실상 똑같나 4. **off-brief** — `00-flow.md`/PRD에서 벗어났나

---

## handoff

- "이거 dear에 넣어줘" → **대상 프로젝트의 CLAUDE.md를 먼저 읽고** 그 컨벤션(FSD/React 등)으로 변환.
- 토큰은 대상 토큰 시스템에 **매핑**한다. 통째 복붙 금지.

## 참고

- `@dsCard` 마커가 호환되므로, 원하면 `/design-sync`로 claude.ai/design에 업로드 가능하다.
